# MVP Definition: synco

**Phase:** Phase 6 — Product (v5, 통합 앱 + AI 개인비서 코어)
**Project:** synco
**Date:** 2026-03-26 (세션 6)
**Confidence:** Medium-High

---

## Core Hypothesis

> **"데이터에 맥락이 없는 것이 문제다. AI가 맥락을 붙이면 가치 없던 데이터가 가치 있게 변하고, CEO는 그 가치에 대해 과금할 것이다"**

이 가설은 4개의 하위 가설로 분해된다:
1. FC와 CEO 모두 AI 개인비서(인맥관리·일정관리·브리핑)에 가치를 느끼고 지속 사용한다
2. CEO가 무료 CRM(인맥/고객 관리)에 가치를 느끼고 가입한다
3. FC가 CEO를 플랫폼에 가입시키는 행동을 한다 (수익 쉐어링 + 브리핑 가치)
4. CEO가 AI가 발견한 매칭 기회에 과금한다 (가치 없던 것에서 가치 발견)

### 왜 이 4가지 가설인가 — 문제 정의와의 연결

synco의 핵심 문제 정의는 "데이터에 맥락이 없다"이다. 이 MVP는 "AI가 맥락을 붙이면 가치가 생기는가?"를 최소 단위로 검증한다:
- **가설 1:** AI 개인비서가 일상적 효용을 제공하여 지속 사용을 만드는지 (리텐션의 기반)
- **가설 2:** CEO도 자기 데이터를 꺼내놓을 동기가 있는지 (데이터 확보의 전제)
- **가설 3:** FC가 이 가치를 CEO에게 전달하는 채널로 작동하는지 (사람이 성사시키는 구조)
- **가설 4:** CEO가 발견된 가치에 돈을 내는지 (수익화 가능성)

> **가설 1이 가설 2~4의 전제다.** 기본 기능(AI 개인비서)이 만족스럽지 않으면 사용자는 앱을 떠나고, 매칭까지 도달하지 못한다. 원초적인 효용이 바탕이 되어야 그 위의 비즈니스 매칭과 가치 창출이 설득력을 가진다.

## 앱 아키텍처: 하나의 앱, 역할 기반 UX

synco는 **앱 하나**다. 가입 시 FC/CEO 역할을 선택하고, 역할에 따라 일부 다른 화면을 보여준다.

```
┌─────────────────────────────────────────┐
│          synco (하나의 앱)               │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │     공통 코어: AI 개인비서         │  │
│  │  인맥관리 · 일정관리 · AI 브리핑   │  │
│  │  연결성 발견 · 엑셀 임포트         │  │
│  │     (FC·CEO 모두 동일, 무료)       │  │
│  └───────────────────────────────────┘  │
│          │                    │         │
│  ┌───────┴──────┐    ┌───────┴──────┐  │
│  │  FC 추가 기능  │    │  CEO 추가 기능 │  │
│  │  CEO 초대     │    │  자원 등록     │  │
│  │  수익 대시보드  │    │  매칭 피드     │  │
│  │  영업 브리핑   │    │  크레딧 시스템  │  │
│  └──────────────┘    └──────────────┘  │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │    하나의 DB (many-to-many)        │  │
│  │  FC↔CEO, CEO↔CEO 관계 그래프      │  │
│  │  → AI가 연결성을 발견하는 기반      │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

**왜 하나의 앱인가:**
- 같은 DB에 FC와 CEO가 있으므로 관계 탐색이 즉각적
- many-to-many 관계 그래프에서 AI가 연결성을 발견 — 앱이 나뉘면 이게 안 됨
- 개발·유지보수 비용 절반 — 사이드 프로젝트에 앱 두 개는 비현실적
- "synco 하나면 된다"가 사용자에게 자연스러움

## Must-Have Features (MVP)

### Tier 0: 공통 코어 — AI 개인비서 (FC·CEO 동일, 무료)

> **이것이 synco의 원동력이다.** 사람들은 앱의 가장 기본적인 원초적 기능이 만족스럽지 않으면 쓰지 않는다. AI 개인비서가 좋아야 사용자가 머물고, 머물어야 매칭까지 간다.

| # | 기능 | 목적 | 검증하는 가설 |
|---|------|------|-------------|
| S1 | **인맥/고객 관리** — 이름, 회사, 업종, 메모, 연락처. AI가 자동 정리·분류·보강 | 카톡+엑셀+전화번호부 대체 | 가설 1: AI 개인비서 효용 |
| S2 | **일정 관리** — 미팅, 약속, 팔로업. AI가 리마인드·스케줄링 | 놓치는 약속 방지, 관계 유지 | 가설 1: AI 개인비서 효용 |
| S3 | **AI 브리핑** — 만날 사람의 최근 뉴스·사업 현황 자동 요약 | 미팅 준비 시간 절약, AI의 즉시 가치 | 가설 1: AI 개인비서 효용 |
| S4 | **엑셀 임포트** — 기존 DB 일괄 등록 | 온보딩 장벽 제거 | 활성화율 |
| S5 | **연결성 발견** — DB many-to-many 관계에서 "이 사람과 저 사람이 연결된다" AI 알림 | 데이터의 숨은 가치 체험 | 가설 1 + 매칭으로의 브릿지 |

> **S5가 핵심 브릿지다.** 인맥관리를 하다 보면 AI가 "사장님이 아는 A와 B를 연결하면 이런 가능성이 있습니다"를 자연스럽게 보여준다. 개인비서 → 매칭으로 넘어가는 순간. 창업자가 직접 체험한 것처럼, 내가 가진 인맥의 연결성을 AI가 보여주는 그 순간이 synco의 가치를 체감하는 순간이다.

### Tier 1: FC 추가 기능 (무료)

| # | 기능 | 목적 | 검증하는 가설 |
|---|------|------|-------------|
| F1 | **CEO 가입 초대** — FC가 자기 CEO에게 가입 링크 발송 | Supply→Demand 전환 | 가설 3: FC→CEO 전환율 |
| F2 | **수익 대시보드** — 내 CEO 고객의 과금 현황, 내 쉐어링 수익 | 수익 쉐어링 동기 가시화 | FC 참여 지속성 |

### Tier 2: CEO 추가 기능 (과금)

| # | 기능 | 목적 | 검증하는 가설 |
|---|------|------|-------------|
| C1 | **사업 자원/유휴 자산 등록** — 설비, 재고, 채널, 관심 분야 | 매칭 기초 데이터 | 가설 2: CEO 데이터 입력 의지 |
| C2 | **AI 매칭 피드** — 블러 처리된 매칭 기회 목록 (무료 월 3건 열람) | 가치 경험 + 과금 유도 | CEO 관심도 |
| C3 | **크레딧 시스템** — 상세 열람/기업명 공개/연결 요청에 크레딧 소모 | 부분유료화 핵심 | 가설 4: CEO WTP |
| C4 | **크레딧 구매** — 추가 크레딧 인앱 구매 | 직접 수익 | CEO 과금 의향 |
| C5 | **연결 요청/수락** — 양측 동의 후 연결 정보 공개 | 매칭 완결 | 매칭 성사율 |

### Tier 3: 플랫폼

| # | 기능 | 목적 |
|---|------|------|
| P1 | **AI 매칭 엔진** — 사용자 프로필 간 시너지 분석 + 매칭 점수 | 핵심 가치 |
| P2 | **프로필 완성도 점수** — 80%+ 시 보너스 크레딧 | 데이터 입력 인센티브 |
| P3 | **관리자 대시보드** — 가입 수, 매칭 수, 과금 현황 | 운영 모니터링 |
| P4 | **역할 기반 라우팅** — 가입 시 FC/CEO 선택 → 역할별 화면 분기 | 하나의 앱, 두 가지 경험 |

## 사용자 여정: AI 개인비서에서 매칭까지

```
[Week 1] AI 개인비서로 진입
  "인맥 정리해볼까" → 엑셀 임포트 / 수동 등록
  → AI가 자동 정리·분류 → "오, 편하네"
  → 일정 등록 → AI가 리마인드 → "이거 괜찮은데"

[Week 2-3] AI 브리핑으로 가치 체험
  미팅 전 AI 브리핑 확인 → "이런 것도 알려주네"
  → 관계 유지 리마인드 → "오래 연락 안 한 사람 챙겨야지"
  → 기본 기능 만족 → 지속 사용 정착

[Week 4+] 연결성 발견 → 매칭으로 자연 전환
  AI가 "사장님이 아는 A의 유휴 설비 + B의 수요 = 거래 가능" 알림
  → "이런 연결이 있었어?" → 매칭 피드 탐색 시작
  → 블러 해제 → 크레딧 결제 → 연결 요청 → 비즈니스 매칭
```

> **핵심:** 매칭을 팔지 않는다. 개인비서를 쓰다 보면 자연스럽게 매칭 가치를 발견하게 된다. 이 순서가 뒤집히면 실패한다.

## Success Criteria

### Tier 0 (AI 개인비서) — 모든 것의 기반

| 지표 | 목표 | 기간 | Fail 기준 |
|------|------|------|----------|
| 총 가입자 (FC+CEO) | 50명+ | Month 1 | <20명 |
| **주 3회+ 활성 사용** | **50%** | **Month 2** | **<25% → 개인비서 가치 부족** |
| AI 브리핑 조회 | 주 100회+ | Month 2 | <30회 |
| 일정 등록 활성 비율 | 40%+ | Month 2 | <15% |

> **Kill Signal (Tier 0):** Month 2까지 활성 사용률 25% 미만 → AI 개인비서 가치가 부족. 매칭 이전에 기본 기능 재설계 필요.

### Tier 1+2 (매칭·과금) — Tier 0 위에 쌓이는 것

| 지표 | 목표 | 기간 | Fail 기준 |
|------|------|------|----------|
| CEO 가입 (FC 경유) | 500명 | Month 3 | <100명 |
| CEO 프로필 완성 (80%+) | 40% | Month 3 | <15% |
| 크레딧 사용 (무료분) | 100건+ | Month 4 | <20건 |
| **유료 크레딧 구매** | **10명+** | **Month 5** | **<3명 → 피봇 검토** |
| 매칭 연결 성사 | 5건+ | Month 5 | <1건 |

> **Kill Signal (Tier 1+2):** Month 5까지 유료 크레딧 구매 CEO 3명 미만 → CEO WTP 가설 기각, 수익 모델 피봇 필요

## Flags

**Red Flags:** None

**Yellow Flags:**
- AI 개인비서 품질이 리텐션을 좌우 — 초기 AI 브리핑/일정 기능의 완성도가 낮으면 Tier 0에서 실패
- MVP 스코프가 공통 코어 + 양면 역할이라 개발량 상당 — Tier 0 먼저 완성 후 Tier 1~2 순차 릴리스 고려
- AI 매칭 엔진 품질이 초기 데이터 부족으로 낮을 수 있음 — 초기에는 수동 큐레이션 보완 고려
- 모바일 웹 MVP → 50대 CEO에게 UX 장벽 가능 — 카카오톡 미니앱 등 대안 검토
