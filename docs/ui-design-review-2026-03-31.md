# synco UI 디자인 리뷰 (2026-03-31)

## 전체 디자인 점수: 7.2 / 10

synco는 깔끔하고 기능적인 CRM 인터페이스를 갖추고 있으며, 전반적으로 일관된 디자인 언어를 유지하고 있습니다. Pretendard 폰트, 보라색(#5B6ABF) 기반 컬러 시스템, 그리고 HTMX 기반 SPA 느낌의 네비게이션은 잘 구현되어 있습니다. 다만 두 개의 별도 인터페이스(연락처 CRM vs 후보자 검색)가 디자인 언어 면에서 불일치를 보이며, 몇 가지 시각적 문제와 UX 개선 여지가 있습니다.

---

## 페이지별 상세 리뷰

### 1. 로그인 페이지

**점수: 8/10**

- **시각적 위계**: 명확함. synco 로고 + 태그라인 + 카카오 로그인 버튼이 수직 중앙 정렬로 깔끔하게 배치
- **레이아웃**: 모바일/데스크톱 모두 중앙 정렬 유지, 반응형 잘 작동
- **컬러**: 카카오 노란색 버튼이 브랜드 가이드라인에 맞게 적용됨
- **한국어 텍스트**: "비즈니스 인맥을 AI가 관리합니다" - 적절한 톤

**문제점:**
- 🟢 Minor: 이용약관/개인정보처리방침 링크의 터치 타겟이 작음 (텍스트 크기 기준 44px 미달 가능성)
- 🟢 Minor: 로고에 별도 그래픽 요소 없이 텍스트만 사용 - 브랜드 인지도 약함

---

### 2. 역할 선택 페이지

**점수: 7.5/10**

- **시각적 위계**: "어떤 역할로 사용하시겠습니까?" 질문이 명확
- **카드 레이아웃**: FC(설계사)와 CEO(대표) 두 옵션이 카드 형태로 잘 구분됨
- **아이콘**: SVG 아이콘 사용 (규칙 준수)

**문제점:**
- 🟡 Warning: "시작하기" 버튼이 비활성화 상태처럼 보임 (회색 톤) - 역할 미선택 시 비활성화인지 항상 비활성화 스타일인지 구분 어려움
- 🟢 Minor: 카드 선택 시 시각적 피드백(선택 상태 강조)이 스크린샷에서 불명확

### 3. 연락처 목록 (Contact List) - 메인 CRM 화면

**점수: 7.5/10**

- **시각적 위계**: 페이지 제목 "연락처" + 필터 버튼 + 검색창 + 리스트 구성이 논리적
- **레이아웃**: 데스크톱에서 사이드바(264px) + 메인 콘텐츠 잘 분리
- **카테고리 필터**: 전체/제조/유통/IT/서비스/금융 - 필(pill) 형태 버튼으로 직관적
- **리스트 아이템**: 이름, 회사명, 업종, 위험도(색상 원), 감정(이모지) 표시
- **빈 상태**: IT 카테고리 필터 시 "아직 등록된 연락처가 없습니다" + "연락처 추가하기" CTA 잘 처리됨
- **모바일**: 하단 네비게이션 바 적절히 표시, 사이드바 숨김

**문제점:**
- 🔴 Critical: **이모지를 상태 아이콘으로 사용** (😊😐😟, 🟡🔴⚪) - UI/UX 규칙 위반. 이모지는 플랫폼마다 렌더링이 다르고 디자인 토큰으로 제어 불가. SVG 아이콘 또는 색상 도트로 교체 필요
- 🟡 Warning: 리스트 아이템 간 정보 밀도가 데스크톱에서 낮음 - 카드 사이 간격이 넓어 한 화면에 보이는 정보량 제한적
- 🟡 Warning: 데스크톱에서 연락처 카드의 우측 영역 활용도가 낮음 - tier_label과 이모지만 표시되어 공간 낭비
- 🟢 Minor: 검색창 placeholder "이름 또는 회사명으로 검색" 적절하나, 검색 아이콘이 없음

### 4. 연락처 상세 (Contact Detail)

**점수: 7/10**

- **시각적 위계**: 이름 > 회사정보 > AI 브리핑 > 유사 고객 > 접점 기록 순서 논리적
- **AI 브리핑 섹션**: 보라색 배경의 카드로 시각적 구분 잘 됨
- **유사 고객**: 카드 형태로 관련 연락처 표시
- **접점 기록**: 타임라인 형태로 날짜별 정리

**문제점:**
- 🟡 Warning: 상단 정보 영역에서 "미분석 (18)"이라는 숫자의 의미가 직관적이지 않음
- 🟡 Warning: "메모"와 "미팅" 탭 버튼이 시각적으로 평범하여 주요 행동 유도(CTA) 역할을 못함
- 🟡 Warning: 접점 기록의 긴 텍스트(전화번호, 이메일 등 혼합)가 모바일에서 가독성 저하
- 🟢 Minor: "편집" 버튼이 우측 상단에 작게 위치 - 발견성 낮음

---

### 5. 연락처 추가 폼 (Contact Create)

**점수: 7.5/10**

- **레이아웃**: 세로 스택 폼, 각 필드에 라벨 명확히 표시
- **필수 항목**: 이름 필드에 빨간 별표(*) 표시 - 규칙 준수
- **셀렉트박스**: 업종/지역/매출규모에 "선택하세요" placeholder 적절

**문제점:**
- 🟡 Warning: 전화번호 필드의 placeholder가 "010-0000-0000"인데, input type이 `tel`인지 확인 필요 (모바일 키보드 최적화)
- 🟡 Warning: 폼 필드들이 전부 full-width로 되어 있어 데스크톱에서 시각적으로 넓음 (max-width 제한 또는 2-column 레이아웃 고려)
- 🟢 Minor: 드롭다운이 기본 브라우저 `<select>` 스타일로, 커스텀 디자인과 일관성 부족

### 6. 전체 일정 (Meeting List)

**점수: 8/10**

- **시각적 위계**: 날짜별 그룹핑 (4월 3일 금요일, 4월 9일 목요일) 잘 구성
- **카드 디자인**: 미팅명, 참석자, 시간, 장소 정보가 깔끔하게 배치
- **시간 표시**: 우측 정렬로 시간(11:00 ~ 12:00)이 명확
- **FAB 버튼**: 우측 상단 "+" 버튼으로 미팅 추가 직관적
- **모바일**: 하단 네비게이션 + 카드 레이아웃 적절

**문제점:**
- 🟡 Warning: 미팅이 2개뿐인 상태에서 빈 공간이 매우 넓음 - 빈 공간에 대한 시각적 처리 부재
- 🟢 Minor: 장소에 이모지(📍) 사용 - SVG 로케이션 핀으로 교체 권장

---

### 7. 미팅 상세 (Meeting Detail)

**점수: 7/10**

- **정보 표시**: 날짜, 시간, 장소가 아이콘과 함께 깔끔하게 나열
- **참석자**: 아바타 + 이름 + 회사명 카드 형태로 표시

**문제점:**
- 🔴 Critical: "수정"과 "취소" 버튼이 동일한 시각적 무게를 가짐 - "취소"는 destructive action으로 빨간색/경고 스타일 필요하지만, 흰 배경에 빨간 텍스트만 사용됨. "수정"도 흰 배경 버튼으로 primary action처럼 보이지 않음
- 🟡 Warning: "삭제" 텍스트가 하단에 작게 표시됨 - destructive action이 일반 텍스트 링크 형태여서 실수로 클릭 가능성 있으나, 위치상 분리는 적절
- 🟡 Warning: 미팅 메모 영역에 "메모를 남겨주세요" + "메모 작성하기" 버튼이 있으나, 빈 상태의 시각적 처리가 단조로움

---

### 8. 미팅 등록 폼 (Meeting Create)

**점수: 7/10**

- **레이아웃**: 연락처 선택 > 제목 > 날짜 > 시작/종료 시간 > 장소 순서 논리적
- **시간 필드**: 시작/종료가 가로로 나란히 배치 - 공간 효율적

**문제점:**
- 🟡 Warning: 날짜 필드가 "mm/dd/yyyy" 영어 형식 - 한국 사용자에게 "yyyy년 mm월 dd일" 형식이 더 적합
- 🟡 Warning: "저장" 버튼이 우측 상단에만 있고 폼 하단에는 없음 - 긴 폼에서 스크롤 후 저장하려면 상단으로 올라가야 함
- 🟢 Minor: 연락처 셀렉트박스가 기본 브라우저 스타일

---

### 9. 매칭 페이지 (Intelligence Matches)

**점수: 7.5/10**

- **빈 상태**: 체인 링크 아이콘 + "연락처가 더 쌓이면 AI가 연결을 찾습니다" + "연락처 추가하기" CTA - 적절한 빈 상태 처리
- **메시지 톤**: 한국어 존대말 일관적

**문제점:**
- 🟢 Minor: 빈 상태 메시지가 약간 모호함 - "몇 명 이상" 등 구체적 기준 제시하면 더 좋음

### 10. 설정 페이지 (Settings)

**점수: 8/10**

- **정보 그룹핑**: 내 정보 / 알림 / 앱 정보 세 섹션으로 카드 분리 - 명확한 구조
- **토글 스위치**: 일정 알림, AI 브리핑, 매칭 알림 - 보라색 활성 토글로 시각적 일관성
- **로그아웃**: 하단에 빨간색 텍스트로 분리 - destructive action 분리 규칙 준수
- **모바일**: 카드 레이아웃 적절, 하단 네비게이션 바 표시

**문제점:**
- 🟢 Minor: "소속"과 "전화번호"가 "-"로 표시 - "미등록" 등 의미 있는 텍스트로 교체 권장
- 🟢 Minor: 이용약관/개인정보처리방침에 chevron(>) 아이콘 있으나 터치 타겟 확인 필요

### 11. 후보자 검색 (Candidate Search) - 별도 인터페이스

**점수: 5.5/10**

- **기능성**: 카테고리 필터(21개 직군) + 후보자 리스트 + AI 검색 FAB
- **데이터 표시**: 이름, 직위, 경력 년수, 카테고리, 학력, 경력사항

**문제점:**
- 🔴 Critical: **연락처 CRM과 완전히 다른 디자인 언어** - 사이드바 없음, 상단 카테고리 탭이 가로 스크롤, 전체적인 레이아웃 구조가 다름. 동일 앱 내에서 두 개의 다른 앱처럼 느껴짐
- 🔴 Critical: **네비게이션 단절** - 후보자 검색 페이지에서 연락처/매칭/설정으로 돌아갈 수 있는 네비게이션이 없음. 사이드바도 하단 네비게이션도 표시되지 않음. synco 로고 클릭 시 다시 후보자 페이지로 감
- 🟡 Warning: 카테고리 탭이 영어(Accounting, EHS, Engineer...)로 되어 있어 한국어 기반 앱과 일관성 없음
- 🟡 Warning: 리스트 아이템의 정보 밀도가 높으나 시각적 구분(카드 경계, 배경색 등)이 약함 - 구분선(border-bottom)만으로 항목 구분
- 🟡 Warning: 스크롤 시 상단 카테고리 필터가 고정되지 않아 긴 리스트에서 카테고리 변경하려면 최상단으로 스크롤해야 함
- 🟡 Warning: AI 검색 FAB 버튼이 우측 중앙에 고정 - 스크롤 콘텐츠를 가릴 수 있음
- 🟢 Minor: "후보자 검색"과 "전체 후보자 56명" 텍스트의 타이포그래피 위계가 약함

### 12. 후보자 상세 (Candidate Detail)

**점수: 6/10**

- **정보 구조**: 기본 정보 > 핵심 요약 > 경력 > 주요 성과 > 역량 키워드 순서

**문제점:**
- 🔴 Critical: 전체적으로 스타일링이 미흡 - 연락처 상세 페이지와 비교했을 때 디자인 완성도가 현저히 낮음
- 🟡 Warning: 텍스트가 한 덩어리로 나열되어 시각적 구분이 어려움 - 카드/섹션 분리 필요
- 🟡 Warning: 경력 정보가 텍스트 블록으로 표시되어 스캔하기 어려움 - 구조화된 레이아웃(회사명/직위/기간 분리) 필요
- 🟡 Warning: 역량 키워드가 단순 텍스트 나열 - 태그(chip/badge) 형태가 더 적합

---

### 13. 엑셀 임포트 페이지

**점수: 8/10**

- **레이아웃**: 파일 업로드 영역(점선 박스) + AI 분류 안내 + 분류 시작 버튼 순서 직관적
- **드래그앤드롭 영역**: 점선 테두리 + 업로드 아이콘 + "파일을 선택하세요" 안내 깔끔
- **AI 기능 안내**: "AI가 자동으로 분류합니다" + 보조 설명이 보라색 배경 카드로 강조

**문제점:**
- 🟡 Warning: "AI 분류 시작" 버튼이 비활성화 상태(회색)인데 비활성화 이유가 명시되지 않음 (파일 미선택 때문이지만 시각적 피드백 부족)
- 🟢 Minor: 파일 제한사항 "XLSX 파일 (최대 5MB, 1,000건 이하)"이 작은 텍스트로 표시

---

### 14. 사이드바 네비게이션

**점수: 7.5/10**

- **구성**: 홈 / 연락처 / 매칭 / 설정 4개 항목 - 간결
- **아이콘**: SVG 아이콘 사용 (규칙 준수), 스트로크 스타일 일관
- **활성 상태**: 보라색 배경 + 보라색 텍스트로 현재 위치 표시
- **모바일**: 하단 네비게이션으로 전환 (4개 항목, 5개 제한 준수)

**문제점:**
- 🟡 Warning: "홈" 클릭 시 `/candidates/` (후보자 검색)로 이동하여 사이드바가 사라짐 - 사용자가 CRM 영역으로 돌아오기 어려움
- 🟡 Warning: 미팅 페이지가 사이드바에 없음 - 연락처 상세에서만 접근 가능하거나 직접 URL 입력 필요
- 🟢 Minor: 네비게이션 항목 간 간격(space-y-1)이 약간 좁음

## 카테고리/리스트 표시 방식 대안 제안

### 연락처 목록 - 현재: 카드 리스트

**현재 방식**: 각 연락처가 전체 너비 카드로 세로 나열

**대안 1: 컴팩트 리스트 (권장)**
- 카드 높이를 줄이고, 정보를 한 줄로 압축 (이름 | 회사 | 업종 | 상태)
- 장점: 한 화면에 더 많은 연락처 표시, 스캔 속도 향상
- 적용: 데스크톱에서 테이블에 가까운 밀도, 모바일에서는 현재 카드 유지

**대안 2: 그룹핑**
- 위험도(위험/주의/양호/미분석) 또는 업종별로 섹션 구분
- 장점: 우선순위 기반 작업 가능
- 적용: 현재 필터 버튼과 병행 가능

### 후보자 검색 - 현재: 단순 리스트

**대안 1: 카드 그리드 (2-3열, 권장)**
- 각 후보자를 카드로 표시, 핵심 정보(이름, 직군, 경력, 현 직장) 구조화
- 장점: 비교 용이, 시각적 구분 명확
- 적용: 데스크톱 3열, 태블릿 2열, 모바일 1열

**대안 2: 테이블 뷰 (데스크톱)**
- 정렬 가능한 열 헤더 (경력년수, 카테고리 등)
- 장점: 대량 데이터 비교에 최적
- 적용: 데스크톱 전용, 모바일은 카드 뷰

## 타이포그래피 분석

- **폰트**: Pretendard - CDN에서 로드, 한국어 앱에 적합한 선택
- **기본 크기**: body 14px~16px 범위로 적절
- **위계**: display(28px), heading(20px), subheading(16px), micro(10px) 정의됨
- **line-height**: 1.2~1.4 범위 - body 텍스트에서 1.5로 올리면 가독성 향상

**개선점:**
- micro(10px)는 최소 크기 규칙(12px) 미달 - 12px로 상향 권장
- 후보자 검색 페이지에서 타이포그래피 위계가 약함

---

## 컬러 시스템 분석

- **Primary**: #5B6ABF (보라색) - 일관되게 사용됨
- **Primary Light**: #E8EAFF - 활성 네비게이션, AI 브리핑 배경에 사용
- **Gray Scale**: Tailwind 기본 gray 계열 사용
- **위험도 컬러**: 이모지로 대체되어 시스템 컬러로 정의되지 않음

**개선점:**
- 위험도에 대한 시맨틱 컬러 토큰 정의 필요 (danger: red, warning: amber, safe: green)
- 이모지 대신 색상 도트 + 텍스트 라벨 조합 사용

---

## 반응형 디자인 분석

- **브레이크포인트**: lg(1024px) 기준으로 사이드바/하단 네비게이션 전환 - 적절
- **모바일**: 하단 네비게이션 4개 항목 (홈, 연락처, 매칭, 설정) - 5개 제한 준수
- **safe-area**: CSS에서 `env(safe-area-inset-bottom)` 처리됨
- **viewport**: `viewport-fit=cover` 적용됨

**문제점:**
- 후보자 검색 페이지는 반응형이지만 사이드바/하단 네비 체계 밖에 있음
- 데스크톱 max-width가 `max-w-md`(모바일)에서 `lg:max-w-7xl`로 점프 - 태블릿 사이즈에서 갑자기 넓어질 수 있음

---

## 접근성 분석

**양호:**
- `focus-visible` 스타일 정의됨 (2px solid #5B6ABF)
- `prefers-reduced-motion` 미디어 쿼리 적용됨
- 네비게이션에 `aria-label="사이드바 네비게이션"` 있음
- `lang="ko"` 설정됨

**개선 필요:**
- 이모지 아이콘에 aria-label 없음 (스크린리더가 "웃는 얼굴" 등으로 읽음)
- 폼 에러 상태에 aria-live 적용 확인 필요
- 색상만으로 상태를 전달하는 부분이 있음 (위험도 원 색상)

### P1 - 높은 우선순위

3. **이모지를 SVG 아이콘/컬러 도트로 교체** - 위험도, 감정, 장소 등 모든 시스템 아이콘

### P2 - 중간 우선순위

6. **후보자 검색 디자인을 CRM과 통합** - 공통 레이아웃 시스템 사용
7. **후보자 상세 페이지 디자인 개선** - 카드/섹션 분리, 키워드 태그화

### P3 - 낮은 우선순위
