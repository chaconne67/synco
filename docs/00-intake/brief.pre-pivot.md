## 문제 정의

금융GA(General Agency) 소속 영업자들이 법인 대표 DB를 대량 보유(1인당 1,000~3,000명)하고 있으나, 1회성 보험영업 후 데이터가 사장되는 구조.

- 보험은 재구매율 ≈ 0 → 고객관리 동기 자체가 부재
- 영업 방식: 카톡으로 연락처 수령 → 미팅 → 보험 제안 → 끝 (원시적, 비체계적)
- 고객(기업 대표)은 보험 제안을 싫어함 → 양측 모두 불만족
- 소중한 대면 채널 + 고객 데이터가 완전히 활용되지 않고 있음

## 솔루션

### 1단계: AI 영업 비서 앱
- OpenClose 스타일의 개인 AI 비서
- 고객/기업 정보 자동 수집 (재무정보, 뉴스, 기업현황 등)
- 미팅 준비 (얘기거리, 미팅 주제 사전 제공)
- 팔로우업, 투두리스트 자동 관리
- 미팅 스케줄링, 이메일 자동 발송
- GA 측 어플리케이션에서 미팅 관리

### 2단계: AI 비즈니스 매칭 엔진
- Many-to-Many 그래프DB로 고객 간 관계 구축
- AI가 고객 간 시너지/비즈니스 기회 상시 발굴
- "보험 팔러 왔다" → "비즈니스 기회를 가져왔다"로 패러다임 전환
- AI가 발굴+제안서 작성, 사람이 연결 구체화
- 예시: 패션회사(여성 고객 2~3만) ↔ 화장품회사(재고 보유) → 할인 행사 매칭
- 예시: 제조공장 필요 부품 → 공급 파트너 연결

### 핵심 인사이트
기존 CRM은 "쓸 이유가 없는 엑셀 시트"에 불과. AI 에이전트가 새로운 수익 기회를 자동 창출하면 CRM을 반드시 써야 하는 강력한 동기가 생김. 인공지능 시대에 가장 중요한 것은 데이터이며, 특히 대면 인상/느낌 같은 데이터는 가치가 매우 높음.

## 창업팀

### 창업자 1 (본인)
- 보험영업 8년 경력 → 영업자 습성/어려움 체감적 이해
- 독학 풀스택 개발 7년차 (투자/자산관리 프로그램 개발 경험)
- 기획 + 개발 동시 수행 가능
- 역할: 기술 + 전략

### 공동창업자
- 현업 금융 영업자
- 2개 GA와 이미 계약 체결, 영업 진행 중
- 본인 보유 DB: 3,000~4,000명
- 역할: 영업 + 현장 + 네트워크

## 시장 컨텍스트

- GA 1곳 평균 영업자: 20~50명, 대형은 수백 명
- 영업자 1인당 DB: 1,000~3,000명
- 협의된 GA: 영업자 ~30명, DB 5만~10만 추정
- 타겟 고객: 법인 대표이사 (직접 전화/방문 가능한 실제 네트워크)

## 수익 모델

- **1차:** 비즈니스 매칭 성사 시 수수료
- **향후 확장:** 자산관리 비즈니스 연결, 금융업 연결, 기타 비즈니스 매칭 수수료

## 경쟁 우위 (Unfair Advantage)

1. 대면 채널 보유 — 단순 DB가 아니라 전화/방문/제안 가능한 실제 네트워크
2. 영업자 도메인 이해 — 직접 경험에서 오는 습성/어려움/동기 체감적 이해
3. 기술 + 도메인 결합 — 기획-개발 원스톱, 빠른 이터레이션 가능
4. 즉시 검증 가능 — 공동창업자 DB 3,000~4,000명으로 프로토타입 테스트

## 검증 상태

- 1개 GA 구두 협의 완료 (DB 활용 + 앱 개발 합의)
- 기업 대표들 대면 확인: 보험 제안은 불편해하나 비즈니스 매칭은 긍정적
- 영업자들: 쓸만한 CRM 부재, 기존 CRM은 "일이 되어버림"

## 마일스톤

- 6개월: MVP 완성 + 5개 GA 협업 구축
- 12개월: 20개 이상 GA 협업 구축

## 제약 사항

- 풀타임 불가, 사이드 프로젝트로 시작
- 자금/런웨이 없음, 생업 병행 필수
- AI 코딩 도구로 개발 속도 확보 예정

## 주요 리스크 & 불확실성

- **개인정보 보호/보안:** 고객 DB 활용 시 정보이용동의 필수
- **시간 제약:** 사이드 프로젝트 + 생업 병행
- **자금 부족:** 외부 투자 없이 자체 부트스트랩
- **규제:** 금융업 관련 규제 가능성

## 킬 크라이테리아 (창업자 관점)

- 규제/보안 문제로 비즈니스 모델 자체가 불가능한 경우
- 단, 매칭이 안 되더라도 AI CRM 비서만으로도 인건비 이상 수익 가능하다고 판단
