# 딜메이킹 플랫폼 구조 및 에스크로 리서치

> 조사일: 2026-03-25 | 웹검색 5회 사용

## 1. 성공적인 딜메이킹 플랫폼 사례

### 1-1. Axial (미국 M&A 매칭 플랫폼)

**플랫폼 개요** [Data]
- 미국·캐나다 Lower Middle Market 대상 (매출 $2.5M~$250M, EBITDA $250K~$25M)
- 약 20,000명 회원: PE 500+, 패밀리오피스 250+, 인수형 포트폴리오 기업 1,500+, 독립 스폰서/HNWI 2,000+
- 연간 10,000건 이상 딜이 플랫폼에 등록

**수익 구조** [Data]
- **멤버십 기반 SaaS 모델**: 바이사이드/셀사이드 모두 구독료 납부 (구체적 금액 비공개)
- 거래 성사 시 별도 수수료를 플랫폼이 부과하지 않음 — 어드바이저(중개인)가 성과보수를 수취
- M&A 어드바이저 일반적 수수료: 최소 $150K, Lehman 방식(첫 $2M에 10%, $5M 초과분 ~5%)

**딜 프로세스** [Data]
1. 바이사이드: 상세 인수 기준(산업, 규모, 지역 등) 입력
2. 셀사이드: 회사/거래 정보를 비공개로 등록
3. 알고리즘이 매칭 → 관련성 순위로 랭킹
4. 셀사이드가 선택적으로 바이사이드에 접근 (티저 발송)
5. NDA 자동화, 딜 관리 툴 제공
- **핵심 원칙**: 비공개·기밀 유지 중심 ("privately and confidentially")

**시사점** [Knowledge-Based] (confidence: 0.8)
- B2B 매칭 플랫폼은 거래 자체에서 수수료를 빼기보다 **SaaS 구독** 모델이 더 현실적
- 알고리즘 매칭 + 기밀 유지가 핵심 가치 제안


### 1-2. 매치드 (한국 B2B 매칭)

[Knowledge-Based] (confidence: 0.5)
- 공개된 수수료 구조나 거래 추적 메커니즘에 대한 상세 정보를 웹에서 확인 불가
- 한국의 B2B 매칭 플랫폼은 대부분 **구독형 + 성과보수 하이브리드** 모델 사용
- 거래 추적은 플랫폼 내 메시징/계약 체결 기능으로 관리하는 것이 일반적


### 1-3. Alibaba Trade Assurance (글로벌 B2B 에스크로)

**에스크로 구조** [Data]
- 바이어 결제금을 에스크로 계정에 예치 → 상품 수령·확인 후 공급자에게 지급
- 프로세스: 보증금 결제 → 생산 → 출하 → 통관 → 수령 확인 → 대금 방출

**수수료** [Data]
- 바이어: Trade Assurance 서비스 자체는 무료
- 공급자(셀러): 거래당 2~3% 수수료 (상한 $100~$350, 기본/프리미엄 티어별 상이)
- 결제수단별 추가: 신용카드 2.95%, Western Union $4.9~$45
- 실질적으로 공급자가 수수료를 제품가에 전가하는 경우 다수

**시사점** [Knowledge-Based] (confidence: 0.8)
- B2B 에스크로에서도 **수수료를 셀러에게 부과**하는 것이 표준
- 대규모 B2B 거래에서 에스크로는 신뢰 구축의 핵심 메커니즘


### 1-4. 크몽 / 위시켓 (한국 프리랜서 매칭 + 에스크로)

**크몽** [Data]
- 안전결제(에스크로) 기반 거래
- 판매 수수료 (구간별 차등):
  - 50만원 이하: **20%**
  - 50만원 초과 ~ 200만원 이하: **12%**
  - 200만원 초과: **6%**
- 수수료 = 서비스 이용료 + 결제망 이용료 + 부가세
- 구매자는 수수료 부담 없음

**위시켓** [Data]
- 에스크로 대금 보호 시스템 + 표준 계약서 기반 안심 계약
- 의뢰자(구매자): 수수료 **무료**
- 파트너(공급자) 수수료 (2026.01.30 개정):
  - 프로젝트 단위 외주(도급): 500만원 이상 **15%**, 500만원 미만 **20%**
  - 기간제 프리랜서 채용(상주): **10%**

**시사점** [Knowledge-Based] (confidence: 0.9)
- 양 플랫폼 모두 **구매자 무료 + 판매자 수수료** 구조
- 거래 규모가 클수록 수수료율이 낮아지는 역진적 구조 → 대형 거래 유인
- 에스크로는 플랫폼 이탈(우회 거래) 방지의 핵심 장치


### 1-5. Upwork / Fiverr (글로벌 프리랜서 매칭 + 에스크로)

**Upwork** [Data]
- 고정가 프로젝트: 에스크로에 대금 예치 → 완료 확인 후 지급
- 2025.05 이전: 프리랜서 일률 10% 수수료
- 2025.05 이후: **가변 수수료 0~15%+** (스킬 수요에 따라 AI가 결정)
  - 고수요 스킬: 0% 가능
  - 포화 카테고리: 15% 이상
- 클라이언트: 별도 서비스 수수료 부과 (마켓플레이스 수수료)
- 출금 수수료: 3% (미국 ACH는 1%)

**Fiverr** [Data]
- 프리랜서: 일률 **20%** 수수료
- 구매자: 주문 $100 초과 시 **5.5%**, $100 이하 시 **$5.50 고정**
- Fiverr는 양면(구매자+판매자) 모두에게 수수료 부과

**시사점** [Knowledge-Based] (confidence: 0.85)
- Upwork의 가변 수수료 모델은 공급 과잉 문제 해결 시도 — 혁신적이지만 프리랜서 반발 있음
- Fiverr는 양면 수수료로 높은 테이크레이트(~25%) 달성
- 에스크로는 모든 주요 프리랜서 플랫폼의 기본 인프라

## 2. 에스크로 구현 방식 (한국)

### 2-1. 전자상거래 에스크로 의무 대상 범위

[Data + Knowledge-Based] (confidence: 0.75)
- **전자상거래 등에서의 소비자보호에 관한 법률** 제24조에 근거
- 의무 대상: 연간 거래횟수 또는 거래금액이 일정 기준 이상인 **통신판매업자** 중 비대면 거래
- **B2C 거래**가 주된 적용 대상
- B2B 거래: 법률상 에스크로 의무 대상이 **아님** — 사업자 간 거래는 소비자보호법 적용 외
- 다만 B2B 플랫폼이 자체적으로 에스크로를 도입하는 것은 자유 (법적 의무가 아닌 선택)

### 2-2. PG사 에스크로 서비스

**주요 PG사 현황** [Data]
- 상위 4사(KG이니시스, NHN KCP, 토스페이먼츠, 나이스페이먼츠)가 시장 65~70% 점유
- 에스크로 실시간 계좌이체: **토스페이먼츠**, **KG이니시스**만 지원

**에스크로 작동 방식** [Data]
- 구매자 결제 → PG사가 대금 예치 → 배송/이행 완료 확인 → 판매자에게 지급
- PG사 수수료: 일반 결제 대비 에스크로 수수료가 소폭 높음 (약 3.0~3.5% 수준)

**PG사별 특징** [Data]
| PG사 | 에스크로 지원 | 특이사항 |
|------|-------------|---------|
| 토스페이먼츠 | O (계좌이체) | 개발자 친화적 API, 스타트업에 강점 |
| KG이니시스 | O (계좌이체) | 시장점유율 1위, 대기업 레퍼런스 다수 |
| NHN KCP | O | B2B 서비스·해외 결제에 강점 |
| 나이스페이먼츠 | O | 안정적 시스템 |

### 2-3. 전자금융업 등록 요건 vs PG 위탁

[Knowledge-Based] (confidence: 0.8)

**직접 에스크로 운영 시 (전자금융업 등록)**
- 전자금융거래법에 따라 **전자지급결제대행업** 또는 **결제대금예치업** 등록 필요
- 최소 자본금: 전자지급결제대행업 10억원, 결제대금예치업 20억원
- 금융위원회 등록, 전산설비·보안 요건 충족 필요
- 자금세탁방지(AML) 의무, 이용자 자금 별도 관리 의무
- **스타트업에게는 현실적으로 어려운 진입장벽**

**PG사 위탁 방식 (권장)**
- 기존 PG사의 에스크로 API를 활용하여 결제/예치/지급 처리
- 전자금융업 등록 불필요 — PG사가 라이선스 보유
- 수수료: 거래당 약 3~4% (결제수단·볼륨에 따라 협의)
- 개발 기간: API 연동 기준 2~4주

### 2-4. B2B 거래에서의 에스크로 적용 가능성

[Knowledge-Based] (confidence: 0.75)

**법적 측면**
- B2B 거래에 에스크로 적용 시 법적 금지 사항 없음
- 다만 소비자보호법상 의무 대상은 아니므로 **자발적 도입**
- 세금계산서 발행, 부가세 처리 등 B2B 특유의 정산 이슈 고려 필요

**실무적 고려사항**
- B2B 거래는 단가가 크고 결제 조건이 복잡 (선금/중도금/잔금, NET 30/60 등)
- 기존 PG 에스크로는 대부분 B2C 소액 거래에 최적화 → B2B 대형 거래에는 커스터마이징 필요
- NHN KCP의 B2B 결제 서비스가 상대적으로 적합
- 대안: **가상계좌 + 플랫폼 자체 정산 로직**으로 유사 에스크로 구현 가능 (단, 자금 예치 시 결제대금예치업 등록 이슈 발생 가능)

## 3. 딜 프로세스 설계 모범 사례

### 3-1. 단계별 플랫폼 역할

[Knowledge-Based + Data] (confidence: 0.8)

| 단계 | 플랫폼 역할 | 참고 사례 |
|------|-----------|----------|
| **매칭** | 알고리즘 기반 수요-공급 매칭, 필터링 | Axial (AI 매칭), Upwork (카테고리+스킬 매칭) |
| **소개** | 익명화된 티저/프로필 제공, 관심 표현 기능 | Axial (비공개 티저), 크몽 (포트폴리오) |
| **협상** | 플랫폼 내 메시징, 견적 교환, 조건 협의 | 위시켓 (견적 비교), Upwork (proposal) |
| **계약** | 표준 계약서 제공, 전자서명, 범위·일정 확정 | 위시켓 (표준계약서), Upwork (milestone 설정) |
| **이행** | 진행상황 추적, 마일스톤 관리, 커뮤니케이션 로그 | Upwork (시간추적, 스크린샷), 위시켓 (단계별 확인) |
| **정산** | 에스크로 지급, 수수료 정산, 세금계산서 처리 | 크몽/위시켓 (안전결제), Alibaba (Trade Assurance) |

### 3-2. 분쟁 해결 메커니즘

[Data + Knowledge-Based] (confidence: 0.8)

**주요 플랫폼 분쟁 해결 방식**

| 플랫폼 | 분쟁 해결 | 특징 |
|--------|----------|------|
| Upwork | 중재 센터 → 플랫폼 조정 → 외부 중재 | 에스크로 동결 후 양측 증거 제출 |
| Fiverr | Resolution Center → CS 개입 | 자동 환불 정책 명확 |
| 크몽 | 고객센터 중재 → 검수 기간 내 이의제기 | 3일 자동 구매확정 |
| 위시켓 | 프로젝트 매니저 중재 → 법률 검토 | PM이 중간에서 조정 역할 |
| Alibaba | Trade Assurance 클레임 → 알리바바 판정 | 증거 기반 판정, 바이어 보호 편향 |

**분쟁 해결 설계 원칙**
1. 에스크로 동결: 분쟁 발생 시 즉시 대금 동결
2. 증거 기반: 양측이 커뮤니케이션 로그, 산출물, 계약서 등 제출
3. 단계적 에스컬레이션: 당사자 협의 → 플랫폼 조정 → 외부 중재/법적 절차
4. 시간 제한: 분쟁 제기 기한 설정 (보통 서비스 완료 후 7~30일)
5. 자동 확정: 기한 내 이의 없으면 자동 대금 지급

### 3-3. 거래 추적/검증 방법 (우회 방지)

[Knowledge-Based] (confidence: 0.85)

**우회 거래 방지 전략**

| 방법 | 설명 | 사용 플랫폼 |
|------|------|-----------|
| **에스크로 필수화** | 결제가 플랫폼을 통해야만 거래 보호 적용 | 크몽, 위시켓, Upwork, Fiverr |
| **연락처 마스킹** | 초기 단계에서 이메일/전화번호 비공개 | Axial, Upwork, 위시켓 |
| **메시지 모니터링** | 외부 연락처 교환 시도 감지 (패턴 매칭) | Upwork (경고 표시), Fiverr |
| **부가 서비스 묶기** | 계약서, 분쟁 해결, 세금처리 등 에스크로에 연결 | 위시켓 (표준계약서), 크몽 |
| **비경쟁 조항** | 플랫폼 소개 거래에 대해 일정 기간 직거래 금지 | Upwork (2년 조항), 위시켓 |
| **가치 제공** | 우회 동기 자체를 줄이는 것이 궁극적 해법 | 전 플랫폼 공통 |

**우회 방지 효과 순위** [Knowledge-Based] (confidence: 0.7)
1. **에스크로 + 분쟁 보호**: 가장 강력 — 돈이 묶이면 이탈 어려움
2. **부가 서비스 번들링**: 계약서·세금처리·보험 등을 에스크로와 연동
3. **연락처 마스킹**: 초기 단계에서 효과적이나, 오프라인 미팅 시 우회 가능
4. **메시지 모니터링**: 보조적 수단, 사생활 침해 논란 가능
5. **비경쟁 조항**: 법적 강제력 낮음, 심리적 억제 효과 주로

| Fiverr | 5.5% | 20% | ~25% |

### 딜메이킹 플랫폼 설계 시 핵심 결정사항

1. **수수료 모델 선택**: SaaS 구독 vs 거래 수수료 vs 하이브리드
   - 거래 규모가 크고 빈도가 낮은 B2B → SaaS 구독이 적합 (Axial 모델)
   - 거래 빈도가 높고 규모가 다양 → 거래 수수료가 적합 (크몽/위시켓 모델)

2. **에스크로 구현**: PG 위탁이 현실적 (전자금융업 등록은 자본금 10~20억 필요)
   - B2B 대형 거래 시 가상계좌 + 자체 정산 로직 검토 필요

3. **우회 방지**: 에스크로·분쟁보호·부가서비스를 하나로 묶어 이탈 비용 높이기

4. **분쟁 해결**: 단계적 에스컬레이션 + 에스크로 동결 + 시간 제한 자동 확정
