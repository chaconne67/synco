# 3단계: 워크플로우 연결

> **작성일:** 2026-04-12
> **범위:** 컨택→추천 전환 자동화 + 단계별 안내 강화
> **선행 조건:** 1단계 (RBAC + 온보딩), 2단계 (설정/관리) 구현 완료

---

## 배경

현재 synco의 헤드헌팅 워크플로우는 프로젝트 상세 페이지의 6개 탭으로 구성된다:

```
[개요] [서칭] [컨택] [추천] [면접] [오퍼]
```

각 탭은 독립적으로 동작하며, 단계 간 전환이 수동이다.
특히 **컨택→추천** 전환에서 사용자 경험이 끊긴다:

### 현재 UX (수동 전환)

```
컨택 탭: 후보자 컨택 → 결과 "관심" 입력
    ↓
컨택 탭: "추천 서류 작성 →" 링크 클릭
    ↓
컨택 탭: 추천 폼이 탭 내부에 인라인 렌더링됨
    ↓
추천 서류 작성 → 등록
    ↓
HX-Trigger: submissionChanged → 추천 탭이 백그라운드에서 갱신
    ↓
문제: 사용자는 여전히 컨택 탭에 있음
      → 수동으로 [추천] 탭을 클릭해야 등록 결과를 확인
```

**핵심 문제:** 추천 서류를 등록한 후 사용자가 추천 탭으로 자동 이동하지 않아,
등록이 성공했는지 확인하려면 탭을 수동으로 전환해야 한다.

## 3.1 컨택→추천 자동 전환

### 변경 1: 추천 서류 등록 후 추천 탭으로 자동 전환

현재 `submission_create()` 뷰는 성공 시 `HX-Trigger: submissionChanged`를 반환한다.
이를 변경하여 **추천 탭 전체를 반환**하고, 동시에 **탭바의 활성 탭을 갱신**한다.

```
변경 전:
  POST submission_create → 204 + HX-Trigger: submissionChanged
  → 컨택 탭 내 폼 영역만 갱신

변경 후:
  POST submission_create → 200 + 추천 탭 파셜 반환
  → hx-target="#tab-content" + HX-Trigger: tabChanged
  → 탭 콘텐츠가 추천 탭으로 교체 + 탭바 활성 상태 갱신
```

**구현 방식:**

`submission_create()` 뷰 수정:
- 성공 시 `project_tab_submissions` 파셜을 렌더링하여 반환한다.
- 응답 헤더에 `HX-Retarget: #tab-content`와 `HX-Trigger: tabChanged` 추가.
- `tabChanged` 이벤트에 `{"activeTab": "submissions"}` 데이터 포함.

`detail_tab_bar.html` 수정:
- `tabChanged` 이벤트 리스너 추가.
- 이벤트 수신 시 해당 탭 버튼에 active 클래스 적용.

### 변경 2: 컨택 결과 "관심" 입력 시 추천 유도 배너

컨택 결과를 "관심"으로 저장한 직후, 컨택 탭에 유도 배너를 표시한다.

```
┌─────────────────────────────────────────────────┐
│ ✓ 홍길동님 컨택 결과가 "관심"으로 저장되었습니다.    │
│                                                  │
│   [추천 서류 작성하기 →]                           │
└─────────────────────────────────────────────────┘
```

**구현 방식:**

`contact_update()` 뷰 수정:
- 결과가 `INTERESTED`로 변경된 경우, 응답에 배너 HTML을 포함한다.
- 배너의 "추천 서류 작성하기" 버튼: `hx-get="submission_create?candidate={id}"` + `hx-target="#tab-content"` + `HX-Trigger: tabChanged`.

이 배너는 일회성이다. 탭을 새로고침하면 사라진다.
이미 추천 서류가 있는 후보자에게는 배너를 표시하지 않는다.

### 변경 3: 컨택 예정(RESERVED) → 컨택 등록 유도

서칭 탭에서 "예정 등록"한 후보자가 컨택 탭에 표시된다.
예정 목록의 각 항목에 "컨택 등록" 버튼을 추가한다.

현재 이미 구현되어 있으나 (`contact_reserve()` → 컨택 탭에 예정 목록 표시),
예정 목록에서 컨택 폼으로의 전환이 명시적이지 않다.

변경: 예정 목록 각 행에 "컨택 등록" 버튼을 추가한다.
클릭 시 해당 후보자가 프리필된 컨택 등록 폼을 인라인으로 연다.

## 3.2 퍼널 진행 표시기

### 프로젝트 개요 탭 퍼널 강화

현재 개요 탭에 퍼널(funnel) 시각화가 있다. 이를 클릭 가능하게 만든다.

```
서칭 (42) → 컨택 (18) → 관심 (7) → 추천 (3) → 면접 (1) → 오퍼 (0)
  [탭]       [탭]      [필터]     [탭]      [탭]      [탭]
```

각 단계 클릭 시 해당 탭으로 전환한다:
- "서칭" → 서칭 탭
- "컨택" → 컨택 탭
- "관심" → 컨택 탭 (결과=관심 필터 적용)
- "추천" → 추천 탭
- "면접" → 면접 탭
- "오퍼" → 오퍼 탭

**구현:** 각 퍼널 단계를 `hx-get` 링크로 만들고, `hx-target="#tab-content"` + `HX-Trigger: tabChanged` 적용.

### 탭 뱃지에 "새로운" 표시

탭바의 카운트 뱃지에 변경 표시를 추가한다.
사용자가 마지막으로 해당 탭을 본 이후 항목이 추가되었으면 뱃지 색상을 강조한다.

```
[컨택 18] [추천 3●] [면접 1]
                 ↑ 새 항목 있음 (파란 점)
```

**구현:**
- 세션 스토리지에 `lastViewed_{project_id}_{tab}` 타임스탬프를 저장한다.
- 탭 전환 시 현재 시간으로 업데이트한다.
- 탭바 렌더링 시, 해당 탭의 최신 항목 생성일이 lastViewed보다 이후이면 뱃지에 점 표시.
- 클라이언트 측 JavaScript로 처리한다 (서버 부하 없음).

## 3.3 범위 밖 (이 단계에서 하지 않는 것)

- **프로젝트 상태 자동 전환 로직 변경** — 이미 `services/lifecycle.py`에서 처리 중이며, 현재 로직에 문제가 없다.
- **면접→오퍼 전환 자동화** — 사용 빈도가 낮아 수동 전환으로 충분하다.
- **서칭→컨택 자동 전환** — 서칭 탭에서 "예정 등록" 기능이 이미 이 역할을 하고 있다.
- **드래그 앤 드롭 칸반** — 프로젝트 목록 페이지의 칸반은 별도 이슈.
- **모바일 최적화** — 탭 전환 UX는 데스크탑 우선. 모바일은 현재 동작 유지.
