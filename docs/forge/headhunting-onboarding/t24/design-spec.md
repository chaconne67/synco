# t24: 탭 뱃지 신규(new) 표시

> **Phase:** 3 — 워크플로우 연결
> **선행 조건:** t19 (tabChanged 이벤트 시스템 — tab-navigation.js 및 data 속성)

---

## 배경

t19에서 탭 뱃지 신규 표시의 기반(tab-navigation.js의 DOMContentLoaded 핸들러, data-badge-count/data-latest 속성, sessionStorage 로직)을 이미 구현했다. 이 태스크에서는 해당 기능이 실제로 올바르게 동작하는지 검증하는 테스트를 작성하고 수동 검증을 수행한다.

## 요구사항

### 탭 뱃지 data 속성 검증

- 프로젝트 상세 페이지에서 탭바의 각 버튼에 `data-tab` 속성이 렌더링되는지 확인한다.
- 뱃지 span에 `data-badge-count`와 `data-latest` 속성이 렌더링되는지 확인한다.

### 신규 표시 동작 검증 (수동)

1. 프로젝트에 새 컨택을 추가한다.
2. 다른 탭(예: 개요)으로 이동한다.
3. 컨택 탭의 뱃지에 파란색 테두리(ring)가 표시되는지 확인한다.
4. 컨택 탭을 클릭하면 파란색 테두리가 사라지는지 확인한다.

## 제약사항

- 구현은 t19에서 완료되었으며, 이 태스크는 테스트와 검증만 수행한다.
- t19가 완료되어야 테스트가 통과한다.
