# Task 9: 빈 화면 CTA (역할별 분기)

> **출처:** `docs/forge/headhunting-onboarding/phase1/design-spec-agreed.md`
> **선행 조건:** Task 1, 2 (구현 완료), Task 7 (사이드바 역할별 메뉴 필터링)

---

## 배경

데이터가 없는 빈 화면에서 사용자에게 다음 행동을 안내하는 CTA(Call to Action)를 역할별로 다르게 표시해야 한다. owner에게는 생성/등록 버튼을, consultant에게는 안내 메시지를 보여준다.

---

## 요구사항

### 역할별 빈 화면 CTA

| 화면 | Owner | Consultant |
|------|-------|-----------|
| 대시보드 | "고객사를 등록하고 첫 프로젝트를 시작하세요" + 고객사 등록 버튼 | "배정된 프로젝트가 없습니다. 관리자가 프로젝트를 배정하면 여기에 표시됩니다." |
| 프로젝트 목록 | "새 프로젝트 만들기" 버튼 | "배정된 프로젝트가 없습니다." |
| 고객사 목록 | "첫 고객사를 등록하세요" + 등록 버튼 | "등록된 고객사가 없습니다." |
| 프로젝트 > 컨택 탭 | "후보자를 서칭하고 컨택을 시작하세요" + 서칭 탭 이동 | 동일 |
| 프로젝트 > 추천 탭 | "컨택에서 관심 후보자가 생기면 추천서류를 작성할 수 있습니다." | 동일 |

### 템플릿 분기 패턴

```html
{% if membership and membership.role == 'owner' %}
  <!-- owner용 CTA: 생성 버튼 포함 -->
{% else %}
  <!-- consultant용 CTA: 안내 메시지만 -->
{% endif %}
```

---

## 제약사항

- Task 2의 context processor가 `membership` 변수를 모든 템플릿에 주입하는 것을 전제한다.
- 기존 빈 화면 레이아웃/스타일은 유지하고, CTA 내용만 역할별로 분기한다.
- 프로젝트 컨택/추천 탭의 빈 화면은 역할 무관하게 동일한 메시지를 표시한다.
