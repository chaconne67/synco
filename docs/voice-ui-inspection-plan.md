- **문제점:** 말풍선(채팅) 아이콘 사용 — 음성 검색 기능을 연상시키지 못함
- **aria-label:** "AI 검색 열기" — 음성 검색과 무관한 레이블

### 1.3 현재 모달 UI (chatbot_modal.html)

**구조:**
1. **헤더:** "synco AI 검색" 제목 + 닫기 버튼 (최소화 아이콘)
2. **메시지 영역:** 채팅 형태 (좌측 AI 메시지, 우측 사용자 메시지)
3. **입력 바:** 마이크 버튼 + 텍스트 입력창 + 전송 버튼

**문제점:**
- 전체 레이아웃이 **채팅 앱** 패턴 (예: 카카오톡)과 동일 — 보이스 퍼스트와 거리가 멀다
- 메시지 영역이 대부분의 공간을 차지하고, 마이크 버튼은 입력 바 좌측에 작게 배치
- 최초 진입 시 텍스트 입력창에 포커스가 가도록 설정 (`chat-input.focus()`)
- 음성 기능이 있다는 것을 인지하기 어려움

### 1.4 현재 마이크 버튼 상태 관리

**위치:** `chatbot.js:196-216` (`setMicState` 함수)

| 상태 | 배경색 변경 | 아이콘 | 추가 피드백 |
|------|------------|--------|------------|
| `idle` | `bg-primary` (#4A56A8) | 마이크 SVG | 없음 |
| `recording` | `bg-red-500` | 정지(사각형) SVG | 없음 — 애니메이션, 파형, 타이머 없음 |
| `processing` | `bg-gray-400` | 스피너 | 없음 |

**문제점:**
- **녹음 중 상태 피드백이 극히 빈약:** 색상 변경 + 정지 아이콘만 표시
- 파형(waveform), 펄스(pulse), 타이머 등 시각적 녹음 인디케이터가 전혀 없음
- 녹음 시간 표시 없음 (사용자가 얼마나 녹음되고 있는지 알 수 없음)
- 오디오 레벨 시각화 없음

### 1.5 현재 에러 처리

| 상황 | 현재 처리 |
|------|----------|
| 마이크 권한 거부 | `showToast("마이크 권한을 허용해주세요")` |
| 음성 인식 실패 | `showToast("음성 인식에 실패했습니다. 텍스트로 입력해주세요.")` |
| 검색 오류 | 채팅 영역에 "검색 중 오류가 발생했습니다." 메시지 |

- 에러 메시지가 토스트(3초 후 자동 사라짐)로만 표시 — 놓치기 쉬움
- 마이크 권한 거부 시 복구 가이드(설정으로 이동) 없음

### 1.6 현재 음성 → 결과 플로우

```
마이크 버튼 클릭 → getUserMedia → MediaRecorder.start()
→ (사용자 말하는 중: 정지 아이콘만 표시)
→ 마이크 버튼 다시 클릭 → MediaRecorder.stop()
→ 스피너 표시 → /candidates/voice/ POST (webm 전송)
→ STT 결과 수신 → 채팅 메시지 추가 (🎤 표시)
→ /candidates/search/ POST → 검색 결과 → 후보자 리스트 갱신
```

- 사용자가 **수동으로 녹음을 시작/종료**해야 함 (Press-to-talk)
- 자동 종료(VAD: Voice Activity Detection) 없음
- 녹음 시작/종료의 오디오 피드백(earcon) 없음

## 2. 유사 서비스 리서치 결과

### 2.1 Google 음성 검색 (2025년 리디자인)

**핵심 변경 사항:**
- 기존 4색 점 파형 → **호(arc) 형태 진동 애니메이션**으로 교체
- 중앙에 Google 'G' 로고 배치
- "What's on your mind?" 프롬프트 → 음성 인식 시 실시간 트랜스크립트로 전환
- 마이크 활성화 사운드를 AI Mode와 통일

**synco 적용 포인트:**
- 녹음 중 **진동/펄스 애니메이션** 도입
- 프롬프트 텍스트가 트랜스크립트로 자연스럽게 전환되는 패턴
- 일관된 오디오 피드백(earcon)

**참조:** [9to5Google — Google app redesigning voice and song search](https://9to5google.com/2025/11/07/google-voice-search-redesign/)

### 2.2 ChatGPT 통합 음성 모드 (2025.11)

**핵심 변경 사항:**
- 기존: 별도 화면의 파란 원 애니메이션 → 대화 히스토리/시각적 응답 접근 불가
- 변경: **같은 채팅 화면에서 음성 사용** → 실시간 트랜스크립트 + 이미지/맵 동시 표시
- 음성과 텍스트를 **같은 대화 쓰레드에서 자유롭게 전환**

**synco 적용 포인트:**
- 음성과 텍스트를 분리하지 않고 **하나의 인터페이스에서 통합**하는 패턴
- 별도 화면 전환 없이 인라인으로 음성 입력을 처리
- 실시간 트랜스크립트 표시

**참조:** [TechCrunch — ChatGPT's voice mode is no longer a separate interface](https://techcrunch.com/2025/11/25/chatgpts-voice-mode-is-no-longer-a-separate-interface/)

### 2.3 Apple Siri (iOS 18)

**핵심 패턴:**
- 활성화 시 **화면 테두리 전체에 빛나는 애니메이션** (iOS 18 신규)
- 듣기/처리 중 상태를 **파형(sinewave) 애니메이션**으로 시각화
- 미세 진동(haptic) 피드백으로 상태 전환 알림
- **siriwave.js** 오픈소스 라이브러리로 웹 구현 가능

**synco 적용 포인트:**
- 파형 애니메이션을 CSS/Canvas로 구현하여 녹음 중 상태 표현
- 상태 전환 시 시각적+촉각적 피드백

**참조:** [GitHub — kopiro/siriwave](https://github.com/kopiro/siriwave)

### 2.4 네이버 Green Dot AI 검색

**핵심 패턴:**
- 검색 바 내부에 마이크 아이콘 배치
- 탭하면 음성 검색 전용 화면 오픈
- 음성, 이미지, 음악 검색 등 다중 입력 모드 지원

**synco 적용 포인트:**
- 검색 바 내 마이크 아이콘 인라인 배치 고려
- 멀티모달 검색의 진입점 통합

### 2.5 VUI 디자인 가이드 (Fuselab 2026)

**핵심 원칙:**
- **"한 호흡" 교환:** 상호작용을 짧고 자연스러운 대화 단위로 설계
- **에러 복구 3원칙:** (1) 문맥 기반 재질문, (2) 2회 실패 후 화면 전환, (3) 무한 루프 방지
- **Earcon 패턴:** 듣기 시작=핑, 처리 중=처리음, 성공=경쾌한 차임, 실패=불협화음
- **3초 이상 무음은 시스템 오류로 인식됨** — 반드시 피드백 제공

**참조:** [Fuselab — Voice UI Design Guide 2026](https://fuselabcreative.com/voice-user-interface-design-guide-2026/)

### 2.7 온보딩 패턴

**핵심 기법 (Voicebot.ai 리서치):**
- **컨텍스트 튜토리얼:** 첫 사용 시에만 해당 기능에 대한 안내를 표시
- **진행 표시:** "7개 질문 중 3번째" 같은 진행률 안내
- **멀티채널 보완:** 음성 안내 + 화면 표시를 동시에 활용
- **핵심 원칙:** "온보딩은 미묘하고, 맥락에 맞고, 신규 사용자에게만 제한되어야 한다"

### 3.4 음성 입력 vs 텍스트 입력 UX

| 점검 항목 | 점검 대상 파일 | 현재 상태 | 기대 상태 | 점검 기준 |
|-----------|---------------|----------|----------|----------|
| 주(主) 입력 방식 | `chatbot_modal.html:41-72` | 텍스트 입력이 주, 마이크는 보조 (좌측 작은 버튼) | **마이크가 주 입력** — 중앙 대형 마이크 버튼, 텍스트는 보조 | 마이크 버튼이 시각적으로 가장 두드러지는지 |
| 초기 포커스 | `chatbot.js:26` | 텍스트 입력에 포커스 (`chat-input.focus()`) | 마이크 버튼에 시각적 주목, 텍스트 포커스 제거 | 모달 오픈 시 키보드가 자동 올라오지 않는지 (모바일) |
| 레이아웃 비율 | `chatbot_modal.html:41-72` | 마이크(40px) : 텍스트(flex-1) : 전송(40px) = 마이크가 1/3 크기 | 마이크 버튼 크기 확대, 또는 음성 전용 모드에서 마이크 중앙 배치 | 마이크 버튼의 시각적 가중치(visual weight) |

## 4. 참조 디자인 가이드

### 4.1 보이스 퍼스트 UI의 5대 원칙 (리서치 종합)

1. **마이크가 주인공이다**
   - 진입점(플로팅 버튼)부터 마이크 아이콘 사용
   - 모달 내에서 마이크 버튼이 시각적으로 가장 크고 중앙에 배치
   - 텍스트 입력은 보조 수단으로 축소

2. **상태를 항상 보여준다**
   - idle: 마이크 아이콘 (+ 미세 idle 애니메이션 선택)
   - recording: **펄스 애니메이션** + 색상 변화 + 선택적 파형
   - processing: 스피너 + "인식 중..." 텍스트
   - 전환 사이 오디오 피드백(earcon) 추가

3. **실패를 우아하게 처리한다**
   - 1회 실패: 맥락 기반 재질문 ("다시 한 번 말씀해주시겠어요?")
   - 2회 실패: 대체 수단 명시적 안내 ("텍스트로 입력해보세요")
   - 무한 루프 절대 방지
   - 3초 이상 무응답 금지

4. **첫 사용자를 안내한다**
   - 플로팅 버튼에 원타임 툴팁
   - 모달 내 예시 검색어 칩
   - "마이크를 누르고 말씀해주세요" 가이드 메시지
   - 컨텍스트 기반 튜토리얼 (필요한 순간에만)

5. **멀티모달을 자연스럽게 지원한다**
   - 음성 → 텍스트 → 음성 전환이 끊김 없이 가능
   - 음성 입력 결과를 텍스트로 확인/수정 가능 (트랜스크립트)
   - ChatGPT 통합 모드처럼 같은 대화 쓰레드 내에서 모달리티 전환

### 4.2 녹음 인디케이터 구현 권장 사양

**최소 구현 (필수):**
```
CSS pulse ring 애니메이션
- @keyframes pulse-ring { 0% { transform: scale(1); opacity: 0.8; } 100% { transform: scale(1.5); opacity: 0; } }
- 마이크 버튼 뒤에 ::before 또는 별도 div로 적용
- 색상: red-500/30 (현재 recording 상태 색상 계열)
```

**권장 구현:**
```
CSS pulse ring + 녹음 시간 카운터 + 오디오 earcon
```

**최상 구현:**
```
Web Audio API 기반 실시간 오디오 레벨 시각화 (Canvas/SVG)
+ 펄스 링 + 시간 카운터 + earcon + 실시간 트랜스크립트
```
