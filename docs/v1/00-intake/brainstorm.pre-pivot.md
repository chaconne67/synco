## 변형 탐색

### 변형 1: "AI 영업 비서 먼저" (현재 방향)
GA 영업자용 AI CRM 비서 → 이후 비즈니스 매칭으로 확장
- **매력:** 즉시 가치 제공, MVP 범위 작음, 매칭의 전제조건인 온보딩을 자연스럽게 해결
- **리스크:** CRM 단독 차별화 약함, 매칭까지 모멘텀 유지 필요
- **경쟁:** 범용 CRM과 겹칠 수 있으나 GA 특화로 차별화

### 변형 2: "비즈니스 매칭 먼저" (역순)
DB를 바로 그래프DB화 → AI 매칭 → 수수료
- **매력:** 핵심 가치 바로 검증, 수수료 모델 빠르게 테스트
- **리스크:** 데이터 품질 없이 매칭은 노이즈, 영업자 참여 없이 데이터 갱신 안 됨
- **경쟁:** 새로운 카테고리

### 변형 3: "기업 대표 네트워크 플랫폼" (타겟 전환)
기업 대표가 직접 사용하는 비즈니스 네트워킹 플랫폼
- **매력:** TAM 큼, 네트워크 효과
- **리스크:** 콜드 스타트, MVP 범위 폭발
- **경쟁:** 링크드인, CEO 모임 등

### 변형 4: "GA SaaS" (비즈니스 모델 전환)
GA 본사에 월 구독료 과금 — 영업관리 + AI 비서 + 고객DB 통합
- **매력:** 예측 가능한 MRR, B2B 영업 단순화
- **리스크:** GA 지불 의향 불확실
- **경쟁:** 보험업 특화 SaaS

### 변형 5: "데이터 중개 플랫폼" (큰 그림)
GA들의 DB 통합 → 기업 간 니즈 매칭 플랫폼화
- **매력:** 네트워크 효과 극대화
- **리스크:** 규제 정면 충돌, 복잡도 과대
- **경쟁:** B2B 마켓플레이스

## 창업자 결정: 우선순위

1. **변형 1+2 (동시 진행):** AI 비서 + 비즈니스 매칭은 분리할 수 없는 하나의 문제 → 핵심 MVP
2. **변형 4:** GA SaaS 구독 모델 → 안정적 MRR 확보 수단
3. **변형 3:** 기업 대표 네트워크 플랫폼 → 확장 단계
4. **변형 5:** 데이터 중개 플랫폼 → 궁극적 비전

## Refined Idea

**AI 영업 비서 + 비즈니스 매칭을 하나의 제품으로 동시 개발.** CRM 기능은 매칭을 위한 데이터 수집/관리 인프라이자, 영업자에게 즉시 가치를 제공하는 온보딩 도구. 비즈니스 매칭은 CRM 사용의 궁극적 동기. 이 둘은 분리된 단계가 아니라 하나의 제품의 양면.

향후 GA SaaS 구독 모델을 추가하여 매칭 성사 전에도 매출을 확보하고, 충분한 DB가 쌓이면 기업 대표 네트워크와 데이터 중개 플랫폼으로 확장.
