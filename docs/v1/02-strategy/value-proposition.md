# Value Proposition: synco — AI 개인비서 & 비즈니스 매칭 플랫폼

**Phase:** Phase 4 — Strategy (v5, 통합 앱 + AI 개인비서 코어)
**Project:** synco
**Date:** 2026-03-26 (세션 6)
**Confidence:** Medium-High

---

## 가치 구조: 2-Layer

synco의 가치는 두 단계로 쌓인다. 순서가 중요하다.

### Layer A: AI 개인비서 — 원천 가치 (FC·CEO 공통, 무료)

> **사람들은 앱의 가장 기본적인 원초적 기능이 만족스럽지 않으면 쓰지 않는다.**
> AI 개인비서가 좋아야 사용자가 머물고, 머물어야 Layer B까지 간다.

**FC든 CEO든 동일하게 느끼는 첫 번째 효용:**
- 인맥/고객 관리가 AI 덕분에 쉽고 원활해짐
- 일정/팔로업을 AI가 알아서 챙겨줌
- 만날 사람의 최근 상황을 AI가 미리 정리해줌
- 내가 기존에 억지로 열심히 해야 했던 것들을 자동으로 해줌

이것이 synco를 **처음 쓰게 만들고, 계속 쓰게 만드는 원동력**이다.

### Layer B: 연결성 발견 → 비즈니스 매칭 — 확장 가치 (CEO 과금)

Layer A를 쓰다 보면 AI가 DB의 many-to-many 관계에서 연결을 발견한다:
- "사장님이 아는 A의 유휴 설비 + B의 수요 = 새로운 거래 기회"
- 놀고 있는 유휴 자산의 가치를 증폭시키는 장치

> **Layer A가 바탕이 되어야 Layer B가 설득력을 가진다.** 이 순서가 뒤집히면 실패한다.

## Value Proposition Canvas — CEO 추가 (Demand Side)

### Customer Profile

**Jobs-to-be-done:**
- **Functional:** 유휴 자원을 활용해 새 수익원 발굴, 사업 파트너/거래처 확보
- **Social:** 사업가 동료 네트워크 확장, 업계에서 "연결이 넓은 사람"으로 인정
- **Emotional:** "가진 자원이 더 있는데 제대로 못 쓰고 있다"는 아쉬움 해소

**CEO 추가 Pains:**
1. 사업 파트너를 찾으려면 지인 소개에 100% 의존 → 우연에 좌우
2. 유휴 자원(설비, 재고, 유통채널, 인맥)이 놀지만 활용 채널 없음
3. 네트워킹 모임/BNI는 시간 대비 효율 낮음
4. 설계사 연락 = 보험 제안 → 귀찮음
5. 사업 정보 노출에 대한 우려 (경쟁사, 보안)

### Value Map (CEO 추가 — 비즈니스 매칭)

**Pain Relievers:**
| Pain | Reliever |
|------|----------|
| 지인 의존 우연 매칭 | AI가 데이터 기반으로 체계적 매칭 |
| 유휴 자원 활용 채널 없음 | AI가 유휴 자원의 새로운 활용처를 자동 발견 |
| 네트워킹 시간 낭비 | 앱에서 5분이면 관심 매칭 확인 |
| 설계사 연락 = 보험 | FC가 "비즈니스 기회"를 가져오는 채널로 전환 |
| 정보 노출 우려 | 단계적 공개 (블러 → 크레딧 → 양측 동의 후 연결) |

**Gain Creators:**
| Gain | Creator |
|------|---------|
| 새 비즈니스 발견 | many-to-many 관계에서 AI가 연결 → 유휴 자산 가치 증폭 |
| 시간 절약 | AI가 매칭 발굴 + 제안서 초안 자동 생성 |
| 검증된 파트너 | FC의 대면 관계 기반 → 신뢰도 내재 |
| 리스크 최소화 | 관심 정보에만 과금, 연결 실패 시 크레딧 환불 검토 |

## Value Proposition Canvas — FC 추가 (Supply Side)

### Customer Profile

**FC 추가 Jobs-to-be-done:**
- **Functional:** 새 수익원 확보 (수익 쉐어링)
- **Social:** "보험쟁이"가 아닌 "비즈니스 커넥터"로 인정
- **Emotional:** CEO에게 보험 외 가치를 줄 수 있다는 자부심

**FC 추가 Pains:**
1. CEO에게 보험 외 줄 게 없어서 차단당함
2. 보험 재구매율 ≈ 0 → 신규 개척만 반복

### Value Map (FC 추가)

| Pain | Reliever |
|------|----------|
| 보험 외 줄 게 없음 | CEO에게 비즈니스 매칭 기회 전달 → 연락 명분 |
| 재구매율 0 | CEO 과금 수익 쉐어링 + 딜 중개 수수료 → 기존 고객에서 지속 수입 |

> **FC:** "가치 없던 기업정보가 AI로 맥락을 만나면 무기가 됩니다 — CEO 매칭 수익을 나눠 받는 새 비즈니스"

## 핵심 자산: CEO 직통 DB의 전략적 가치

> **이 비즈니스의 핵심 자산 = GA가 이미 보유한 양질의 CEO 데이터베이스**

### GA의 유휴 자산 → synco의 수익화
- GA/FC의 CEO DB는 보험 판매 후 **추가 가치 = 0** (재구매율 ≈ 0)
- synco가 이 유휴 DB의 잠재력을 수익화 → GA/FC에게 새 수입원 = 윈윈

> **벤치마크:** BNI 한국 2,300명이 리퍼럴로 연간 수천억 거래 창출. synco는 30만 FC 네트워크를 활용하여 이 모델을 10~100배 스케일. [Data + Opinion]

### DB가 돈으로 살 수 없는 이유
1. **CEO 직통 전화번호** — 회사 대표번호가 아닌 개인 번호. 대면 영업에서만 획득
2. **대면 전달 채널(FC)** — 만나서 이야기 전할 수 있는 사람이 이미 존재. 웜 채널

### 활성화된 DB 위의 파생 비즈니스
- **타겟 광고**: 금융사 유튜브 광고 건당 2,000만원+ → CEO 타겟 = 더 높은 단가/전환율
- **로보어드바이저 유통**: CEO = 고자산 고객 → 투자자문 직접 유통 (Layer 3)
- **금융상품 대면 채널**: CEO DB 10만+ 시 금융사 상품 유통
- **B2B 서비스 제휴**: 법무·회계·컨설팅·HR·IT 등 CEO 대상 무한 파생

> **매칭은 DB를 활성화시키는 도구. 활성화된 DB 위에 무한한 비즈니스가 파생된다.**

## Proof Points & Credibility

1. **FC 대면 관계 기반 신뢰** — 콜드 매칭이 아니라 FC가 이미 관계 있는 CEO 간 연결
2. **법인영업 5년+ 인프라** — 전국 GA FC 288,446명의 기존 CEO 직통 DB [Data, FSS]
3. **리멤버 성장 실증** — 유사 B2B 정보 서비스, 매출 684억, EQT 약 5,400억원 인수 [Data]
4. **GA협회 2035 로드맵 정렬** — 금융판매전문회사 전환과 Layer 3 일치 [Data]
5. **CEO 네트워킹 WTP 검증** — 리멤버 Black 30만원/년, BNI 약 135~405만원/년 [Data]
6. **창업자 도메인 복합성** — 보험 8년 + 개발 7년 + 로보어드바이저 운용

## Flags

**Red Flags:** None

**Yellow Flags:**
- CEO Pain "유휴 자원 방치"의 심각도 미검증 — CEO 인터뷰 필요
- "넣으면 나온다" 선순환이 Cold Start에서 작동하려면 초기 데이터 시딩 필요
- 정보 노출 우려는 실제 장벽 — 단계적 공개 + 동의 기반 구조 필수
