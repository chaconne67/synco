**Project:** synco
**Date:** 2026-03-26 (세션 6)
**Confidence:** Medium-High

## Experiment 0: AI 개인비서 효용 검증 (FC 10명 + CEO 10명)

### 가설

> "FC와 CEO 모두 AI 개인비서(인맥관리·일정관리·AI 브리핑)를 주 3회 이상 지속 사용하며, 기존 관리 방식(카톡/엑셀/수첩)보다 낫다고 평가할 것이다."

### 실험 설계

**대상:**
- FC 10명 (공동창업자 GA 소속, Exp 2와 동일 대상 겸용)
- CEO 10명 (공동창업자 DB에서 선정)

**방법:** 2주 실사용 + 사후 인터뷰 (10분)

**프로세스:**

```
Week 1: 온보딩
  - 엑셀/연락처 임포트로 기존 DB 등록 (10분 이내)
  - AI 브리핑 3건 자동 생성 → 카톡/이메일로 전달
  - 일정 1건 등록 유도

Week 2: 자유 사용
  - 사용 로그 추적 (로그인, 데이터 입력, 브리핑 조회, 일정 등록)
  - 중간 리마인드 1회

Week 3: 사후 인터뷰
  Q1. "2주 동안 얼마나 자주 쓰셨어요?" (실제 vs 체감)
  Q2. "기존 방식(카톡/엑셀)과 비교하면?" (1~5점)
  Q3. "가장 유용했던 기능은?" (인맥관리/일정/브리핑/기타)
  Q4. "계속 쓸 의향이 있으세요?" (1~5점)
  Q5. "이거 없으면 불편하겠다 싶은 순간이 있었나요?"
```

### 측정 지표

| 지표 | 측정 방법 | 성공 기준 | Kill 기준 |
|------|----------|----------|----------|
| 주 3회+ 활성 사용 | 로그인+행동 로그 | 50%+ (10명+) | <25% |
| 기존 대비 만족도 | Q2 5점 척도 | 평균 3.5+ | <2.5 |
| 지속 사용 의향 | Q4 5점 척도 | 평균 4.0+ | <3.0 |
| AI 브리핑 조회율 | 자동 생성 대비 조회 | 60%+ | <30% |
| "없으면 불편" 비율 | Q5 정성 응답 | 40%+ | <15% |

### 분석 프레임

```
Strong Signal (Tier 1~2 진행):
  활성 50%+ AND 만족도 3.5+ AND 지속 의향 4.0+

Weak Signal (기능 개선 후 재검증):
  활성 25~50% AND 특정 기능만 사용 → 해당 기능 강화

Kill Signal (Tier 0 실패):
  활성 <25% AND 만족도 <2.5 → AI 개인비서 가치 부족, 기본 기능 재설계
```

### 타임라인

| 일정 | 활동 |
|------|------|
| Day 1-2 | 대상 선정 + 온보딩 |
| Day 3-14 | 2주 자유 사용 + 로그 추적 |
| Day 15-17 | 사후 인터뷰 |
| Day 18 | 결과 분석 + 의사결정 |

**비용:** 5~10만원 (AI API + 기프티콘)
**선행 조건:** MVP Tier 0(AI 개인비서) 최소 기능 구현 완료

> **Exp 0은 Exp 1~4보다 먼저 실행하거나 동시 실행해야 한다.** Tier 0이 실패하면 Tier 1~2 검증의 전제가 무너진다.

## Experiment 1: CEO 관심도 인터뷰 (20명)

### 가설

> "중소기업 CEO는 자사의 유휴 자원(설비, 재고, 유통채널, 인맥)을 활용한 새로운 비즈니스 기회에 관심이 있으며, 이를 AI가 찾아주는 서비스에 월 5~10만원을 지불할 의향이 있다."

### 실험 설계

**대상:** 공동창업자 DB CEO 20명 (업종·규모 다양하게 선정)
- 제조업 5명, 유통/물류 5명, 서비스업 5명, IT/기타 5명
- 매출 10~500억 범위

**방법:** 반구조화 인터뷰 (15분, 전화 또는 대면)

**인터뷰 스크립트:**

```
[도입 2분]
"안녕하세요, 저는 ___입니다. 잠깐 여쭤볼 게 있어서요.
지금 사업하시면서 덜 쓰이는 자원이나 설비가 있으신가요?"

[탐색 5분]
1. "그런 유휴 자원을 다른 사업에 활용해본 적 있으신가요?"
   → Yes: "어떻게 찾으셨어요? 어려운 점은요?"
   → No: "관심은 있으셨어요? 왜 안 해보셨어요?"

2. "다른 업종의 CEO와 사업 협업을 해본 적 있으신가요?"
   → "어떻게 만나셨어요? 결과는요?"

3. "새로운 사업 파트너나 기회를 찾을 때 가장 어려운 점은?"

[솔루션 제안 5분]
4. "만약 AI가 '당신의 유휴 설비로 이런 사업이 가능합니다,
    이 CEO와 협업하면 됩니다'라고 제안한다면 어떠세요?"
   → 관심 수준 5점 척도 (1: 전혀 / 5: 지금 당장)

5. "이런 매칭 정보를 보려면 건당 5,000원이라면?"
   → "월 9.9만원 구독이라면?"
   → "어느 수준이면 쓰시겠어요?"

[마무리 3분]
6. "이런 서비스가 나오면 써보시겠어요?"
7. "주변에 관심 있을 분이 계실까요?"
```

### 측정 지표

| 지표 | 측정 방법 | 성공 기준 |
|------|----------|----------|
| 유휴 자원 보유율 | Q1 응답 | 60%+ (CEO에게 이 문제가 존재) |
| 솔루션 관심도 | Q4 5점 척도 평균 | 3.5+ |
| "지금 당장" 비율 | Q4에서 5점 비율 | 15%+ (3명+) |
| 가격 수용도 | Q5 응답 | 건당 5,000원 수용 50%+, 월 9.9만 수용 30%+ |
| 서비스 사용 의향 | Q6 응답 | 70%+ |
| 추천 의향 | Q7 응답 | 40%+ |

### 분석 프레임

```
Strong Signal (진행):
  - 관심 3.5+ AND 가격 수용 30%+ AND "지금 당장" 3명+

Weak Signal (조건부 진행):
  - 관심 3.0+ BUT 가격 저항 → 가격 재설계 후 재검증
  - 관심 높음 BUT 유휴 자원 보유율 낮음 → 타겟 재정의

No Signal (중단/피봇):
  - 관심 2.5 미만 AND 가격 수용 10% 미만
```

### 타임라인

| 일정 | 활동 |
|------|------|
| Day 1 | CEO 20명 선정 + 인터뷰 일정 잡기 |
| Day 2-4 | 인터뷰 진행 (하루 5~7명) |
| Day 5 | 결과 코딩 + 분석 + 의사결정 |

**비용:** 0원 (시간만)

## Experiment 2: GA 파트너십 경유 FC의 CEO 소개 행동 검증 (10명)

### 가설

> "GA 법인 파트너십을 체결하고 GA가 소속 FC에게 앱 도입을 안내하면, 편입된 FC가 자기 CEO 고객에게 앱을 소개하고 가입시키는 행동을 할 것이다."

### 실험 설계

**대상:** GA 파트너십으로 편입된 FC 10명 (공동창업자 GA 소속)
- 법인영업 경력 3년+ 5명, 1~3년 5명
- 보유 CEO 고객 30명+ 전원

**방법:** 대면 미팅 (30분, 개별 또는 소그룹) — GA가 FC에게 앱 도입을 안내한 후 진행

**미팅 스크립트:**

```
[Phase 1: AI 비서 데모 — 10분]
"GA에서 안내받으셨을 텐데, 이런 도구가 있습니다."
→ 실제 또는 목업 화면으로 시연:
  - 엑셀 고객 DB 임포트 (2분이면 끝)
  - AI가 고객 기업 뉴스 + 재무 요약 생성
  - 미팅 브리핑: "이 고객은 최근 공장 증설, 화재보험 갱신 필요할 수 있음"
"이 도구가 무료라면 쓰시겠어요?"

[Phase 2: 수익 쉐어링 제안 — 10분]
"여기서 한 가지 더 있습니다."
→ 설명:
  - "고객 CEO에게 비즈니스 매칭 서비스를 소개해주세요"
  - "CEO가 가입하고 유료 서비스를 쓰면, 수익의 15%가 매월 입금됩니다"
  - "CEO 10명 Premium 가입 시 월 15만원 추가 수입"
→ 수익 시뮬레이션 표 보여주기

[Phase 3: CEO 소개 행동 의향 확인 — 10분]
1. "지금 CEO 고객 중 이 서비스에 관심 가질 분이 몇 명 정도 계실까요?"
2. "CEO에게 앱을 소개하고 가입 링크를 보내실 수 있는 분은?"
   → 구체적 이름/수 확인
3. "GA에서 안내받았을 때 첫 느낌은? 가장 걱정되는 점은?"
4. "어떤 조건이 추가되면 더 적극적으로 CEO에게 소개하시겠어요?"
```

### 측정 지표

| 지표 | 측정 방법 | 성공 기준 |
|------|----------|----------|
| AI 비서 사용 의향 | Phase 1 반응 | 80%+ (8명+) |
| CEO 소개 행동 의향 | Phase 3 Q2 | 50%+ (5명+) |
| 예상 소개 CEO 수 | Phase 3 Q2 구체적 수 | FC당 평균 3명+ |
| 수익 동기 충분성 | Phase 3 Q4 반응 | "이 정도면 해볼만" 50%+ |
| GA 안내 효과 | Phase 3 Q3 반응 | GA 경유 인지도 + 신뢰 형성 여부 |
| 우려 사항 | Phase 3 Q3 정리 | Top 3 패턴 도출 |

### 분석 프레임

```
Strong Signal: 5명+ "소개하겠다" + FC당 3명+ + GA 안내가 신뢰 형성에 기여
Weak Signal: 3~4명 "소개하겠다" + 우려 사항 명확 → GA→FC 전달 구조 조정 후 재시도
No Signal: "소개하겠다" 2명 이하 → GA 파트너십 인센티브 재설계 또는 FC 채널 전략 근본 재검토
```

### 후속 행동

- **Strong Signal 시:** 바로 실제 링크 발송 요청 → Exp 3으로 연결
- **우려 패턴 발견 시:** 우려 해소 후 2차 미팅 (1주 내)

**비용:** 0원 (대면 시간)

## Experiment 4: 컨시어지 MVP — 수동 매칭 테스트

### 가설

> "CEO에게 AI 기반(실제로는 수동) 매칭 결과를 블러 처리 후 제공하면, 상세 정보 열람에 건당 5,000원을 결제할 것이다."

### 실험 설계

**대상:** Exp 3 등록자 또는 공동창업자 DB CEO 15~20명
- 프로필 작성 완료자만 포함
- 업종 최소 5개 이상 분산

**방법:** 수동 매칭 + 실제 과금 (Wizard of Oz)

**프로세스:**

```
Step 1: CEO 프로필 수집 (구글 폼, 10분)
  - 사업 분야, 매출 규모
  - 보유 자원: 설비, 재고, 유통채널, 인력, 기술
  - 유휴 자원: "덜 쓰이는 것"
  - 관심 분야: 신규 사업, 위탁생산, 공동 마케팅, 인맥 교류 등

Step 2: 수동 매칭 분석 (1~2일)
  - CEO A × CEO B 시너지 분석 (GPT-4o 보조 + 수작업)
  - 매칭 보고서 작성:
    [블러 버전] "OO 업종 기업이 귀사의 유휴 @@설비에 관심이 있습니다.
                예상 시너지: 월 OOO만원 규모. 상세 보기 →"
    [상세 버전] 기업명, 대표 프로필, 구체적 협업 시나리오

Step 3: 블러 매칭 결과 전달 (카톡 또는 이메일)
  - CEO당 3~5건의 블러 매칭 결과 전달
  - "상세 정보를 보시려면 건당 5,000원입니다"
  - 결제 수단: 카카오페이 송금 / 토스 간편결제

Step 4: 결제 후 상세 정보 전달
  - "연결을 원하시면 25,000원입니다"
  - 양측 동의 시 연결 정보 공개

Step 5: 사후 인터뷰 (5분)
  - "매칭 품질은 어떠셨어요?" (1~5점)
  - "이 가격이 적절한가요?"
  - "정기적으로 이런 매칭을 받고 싶으신가요?"
```

### 측정 지표

| 지표 | 측정 방법 | 성공 기준 | Kill 기준 |
|------|----------|----------|----------|
| 프로필 완성율 | 구글 폼 제출 / 요청 | 60%+ | <30% |
| 블러 해제 결제율 | 결제 건 / 매칭 제공 건 | 30%+ | <5% |
| 연결 요청 결제율 | 연결 결제 / 블러 해제 건 | 15%+ | 0건 |
| 매칭 품질 만족도 | 사후 인터뷰 5점 척도 | 3.5+ | <2.5 |
| 정기 이용 의향 | 사후 인터뷰 | 50%+ | <20% |
| **총 결제 금액** | 실제 수금액 | **25만원+** | **5만원 미만** |

### 분석 프레임

```
Strong Signal (MVP 개발 진행):
  - 블러 해제 30%+ AND 연결 요청 2건+ AND 만족도 3.5+
  - → AI 매칭 엔진 개발 착수, 가격 확정

Pivot Signal (모델 조정):
  - 블러 해제 있으나 연결 요청 0건 → 매칭 품질 문제
  - 만족도 높으나 결제 저조 → 가격 재설계
  - 프로필 완성율 낮음 → 입력 UX 재설계

Kill Signal (CEO WTP 가설 기각):
  - 블러 해제 결제 2건 미만 (전체 CEO 15명+ 중)
  - → 과금 모델 근본 재설계 또는 프로젝트 중단 검토
```

### 타임라인

| 일정 | 활동 |
|------|------|
| Day 1-2 | CEO 모집 + 구글 폼 배포 |
| Day 3-5 | 프로필 수집 대기 + 리마인드 |
| Day 6-8 | 수동 매칭 분석 + 보고서 작성 |
| Day 9-10 | 블러 매칭 전달 + 결제 안내 |
| Day 11-14 | 결제 수집 + 상세 전달 + 연결 |
| Day 15 | 사후 인터뷰 + 결과 분석 |

**비용:** 5~10만원 (결제 수수료 + GPT API)

---

## Flags

**Red Flags:** None

**Yellow Flags:**
- Exp 1 인터뷰는 "말"이지 "행동"이 아님 — 관심 표명과 실제 결제는 다르므로 Exp 4까지 가야 진짜 검증
- 컨시어지 MVP의 수동 매칭은 확장 불가능 — AI 자동화 전환 시 품질 하락 리스크 별도 검증 필요 (Exp 6)
- 표본 크기 15~20명은 통계적으로 약함 — 방향성 확인용이지 확정은 아님
