# CEO WTP 검증 설문 v2

**Project:** synco
**Date:** 2026-03-27
**Version:** v2 — 메인사업 B2B 매칭 중심 모델
**Purpose:** CEO가 "AI B2B 거래처 매칭 앱"에 지불 의향이 있는지 검증
**Target:** 중소기업 CEO 30명+ (매출 10~500억, 업종 다양)
**Method:** 온라인 설문 (구글 폼 / 타입폼) + 고관심 응답자 심층 인터뷰

---

## v2 모델 변경 사항

**v1 → v2 핵심 변화:**

| 구분 | v1 (기존) | v2 (변경) |
|------|-----------|-----------|
| 매칭 주력 | 유휴 자원 기반 매칭 | **메인 사업 기반 B2B 거래처 매칭** |
| 유휴 자원 | 핵심 과금 포인트 | 보너스 옵션 (관계 점수 가산) |
| 매칭 점수 | 유휴 자원 일치도 | **메인 사업 거래 가능성** + 유휴 자원 보너스 |
| 핵심 질문 | "유휴 자원 있으세요?" | **"귀사는 무엇을 팔고 무엇이 필요하세요?"** |

> **관계 점수 산정 원칙:**
> 메인 사업 거래 적합도 (70%) + 유휴 자원 매칭 보너스 (30%)
> → 메인 사업 + 유휴 자원 둘 다 맞으면 최고 점수

---

## 설계 원칙

1. **현재 행동(As-Is) → 문제 인식 → 솔루션 반응 → WTP** 순서로 퍼널 구성
2. **"쓰겠다"가 아니라 "얼마를 내겠다"를 묻는다** — WTP가 핵심
3. **메인 사업 매칭이 기본, 유휴 자원 매칭은 추가 옵션** — 거래 가능성이 높을수록 점수 상승
4. **사전 등록(연락처 남기기)이 가장 신뢰할 수 있는 행동 지표**

## Section 6. 솔루션 컨셉 제시 + 반응

> *3분 — 앱 컨셉을 보여주고 반응 측정*

### [컨셉 카드]

> **AI B2B 매칭 앱** — synco는 이렇게 작동합니다:
>
> **① 메인 사업 거래처 매칭 (핵심)**
> AI가 귀사의 주력 사업과 실제로 거래 가능한 기업을 찾아드립니다.
> "귀사 제품을 구매할 바이어", "귀사가 필요한 공급사", "협력 가능한 파트너"를 AI가 발굴합니다.
>
> **② 유휴 자원 활용 (보너스 옵션)**
> 메인 사업 매칭에 더해, 유휴 설비·재고·공간까지 활용할 수 있는 기회를 추가로 제안합니다.
> 메인 거래 + 유휴 자원까지 함께 맞으면 관계 점수가 올라갑니다.
>
> **③ AI 인맥 관리 & 브리핑**
> 거래처 연락처를 한 곳에서 관리하고, 미팅 전 상대방의 최근 사업 현황을 AI가 자동 브리핑합니다.
>
> **④ 신뢰 기반 연결**
> AI가 매칭을 발견하고, 양측을 대면으로 아는 FC(전문 컨설턴트)가 실제 거래로 연결합니다.
>
> *인맥/일정 관리, 메인 사업 등록은 무료.*
> *매칭 상세 정보 열람 및 직접 연결은 유료입니다.*

| 유휴 자원은 보너스로 작동하는가? | Q14+Q15 | 보유율 50%+이나 주요 동기는 아님 확인 |

### Van Westendorp 분석 (Q22~Q25)

```
4개 곡선 교차점:
- PMC (Marginal Cheapness): "싸다" × "적절하다" → 가격 하한
- PME (Marginal Expensiveness): "비싸다" × "적절하다" → 가격 상한
- IDP (Indifference Price): "싸다" × "비싸다" → 최적 가격
- OPP (Optimal Price Point): 최적 수익 극대화 가격

→ 크레딧 단가 / 구독가 적정성 판단
```

### 세그먼트 교차 분석

| 축 | 필터 | 분석 포인트 |
|----|------|-----------|
| 업종별 | Q2 | 제조 vs 서비스 업종별 WTP·니즈 차이 |
| 매출별 | Q3 | 매출 규모 ↔ WTP 상관관계 |
| 메인 수요 有 vs 無 | Q9+Q10 | 거래처 수요 보유자의 WTP 프리미엄 |
| 선호 기능 | Q17 | 메인 매칭 vs 유휴 자원 매칭 선호 세그먼트 |
| 선호 과금 모델 | Q28 | 크레딧 vs 구독 vs 성과기반 세그먼트 크기 |

### 의사결정 매트릭스

```
■ Go (MVP 개발):
  관심도 3.5+ AND 크레딧 WTP 50%+ AND 사전 등록 20%+
  AND "메인 매칭" 1순위 선택 50%+
  → 메인 사업 매칭 중심으로 MVP 개발

■ 조건부 Go (모델 조정):
  관심도 높으나 WTP 낮음
    → 무료 CRM + 광고/데이터 수익으로 피봇 검토
  "유휴 자원" 선호 > "메인 매칭" 선호
    → v1 모델(유휴 자원 중심)로 회귀 검토
  성과기반 선호 강함
    → 딜 중개 수수료 모델 강화
  거래처 탐색 수요 약함
    → CRM 개인비서 단독 상품으로 피봇

■ Kill (CEO WTP 가설 기각):
  관심도 <2.5 AND 유료 의향 <15% AND 사전 등록 <5%
  → 과금 모델 근본 재설계 또는 프로젝트 피봇
```

## 운영 가이드

### 배포
- **구글 폼** 또는 **타입폼** (모바일 최적화 필수)
- 예상 소요: **8~10분**
- 인센티브: 스타벅스 기프티콘 5,000원 (완료자 전원 or 추첨)

### 표본
- 최소 **30명**, 이상적 50명+
- 공동창업자 DB에서 업종/매출 다양하게
- 제조업 + 서비스업 비율 균형 유지 권장 (각 25%+)
- FC 경유 배포 시 "내 설계사가 추천" 바이어스 주의

### 후속 행동

| 조건 | 행동 |
|------|------|
| Q16 4~5점 + Q30 "예" | → **심층 인터뷰 대상** (10명 목표) |
| Q17 "유휴 자원" > "메인 매칭" | → v1 모델 재검토 인터뷰 |
| Van Westendorp 결과 확보 | → **크레딧/구독 가격 확정** |
| 사전 등록 20명+ | → **컨시어지 MVP 대상 풀** |
| Q9+Q10 "지금 당장" 40%+ | → 메인 사업 매칭 MVP 즉시 착수 |

### 설문 한계 (인지 필수)
- 설문은 "말"이다. 실제 결제는 컨시어지 MVP에서 검증해야 한다
- "쓰겠다" 응답의 50~70%만 실제 행동으로 전환되는 것이 일반적
- Van Westendorp은 방향성 지표이지, 가격 확정 도구가 아니다
- **사전 등록(Q30)과 추천 행동(Q31)이 가장 신뢰할 수 있는 행동 지표**
- v2 모델 전환 여부는 Q17 기능 선호 결과가 핵심 판단 근거
