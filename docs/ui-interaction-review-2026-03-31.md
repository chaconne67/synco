## 발견된 문제점

### 🔴 Critical (배포 차단)

#### C1. HTMX 네비게이션 아키텍처 충돌
- **현상**: 후보자 검색 페이지(`/candidates/`)와 CRM 페이지(`/contacts/`, `/meetings/`)가 서로 다른 레이아웃(base template)을 사용하면서 같은 `#main-content` HTMX 타겟을 공유
- **증상**: 사이드바 네비게이션 클릭 시 DOM과 시각적 표시가 불일치. 예를 들어 미팅 리스트가 시각적으로 보이지만 DOM의 `<a>` 링크는 후보자 페이지의 것
- **영향**: 사이드바를 통한 페이지 전환 후 모든 인터랙션이 예측 불가능하게 동작
- **재현**: 후보자 검색 → 사이드바 "연락처" 클릭 → "추가" 버튼 클릭 → 미팅 상세 페이지로 이동됨

#### C2. 인증 보호 불일치
- **현상**: 일부 페이지는 인증 없이 접근 가능하고, 일부는 로그인 필요
- **인증 불필요**: `/candidates/`, `/contacts/`, `/meetings/`, `/contacts/new/`, `/contacts/import/`
- **인증 필요**: `/intelligence/matches/`, `/accounts/settings/`
- **영향**: 민감한 고객 연락처 데이터가 인증 없이 노출. CRM 데이터(`contacts`, `meetings`)는 반드시 인증이 필요한 영역

#### C3. 연락처 검색 시 `hx-include` 오류
- **현상**: 검색 입력창에 `hx-include="[name='industry']"` 속성이 있으나 해당 `name="industry"` 요소가 DOM에 존재하지 않음
- **콘솔 에러**: `The selector "[name='industry']" on hx-include returned no matches!`
- **영향**: 검색 시 업종 필터 상태가 포함되지 않아 필터와 검색이 연동되지 않음

### 🟡 Warning (배포 전 수정 권장)

#### W1. 편집 폼에서 기존 드롭다운 값 미선택
- **현상**: 연락처 편집(`/contacts/{id}/edit/`) 진입 시 업종, 지역, 매출규모 드롭다운이 모두 "선택하세요"로 표시
- **기대**: 기존에 저장된 값이 pre-selected 되어야 함
- **영향**: 사용자가 편집 시 기존 값을 모르면 잘못된 값을 저장할 수 있음

#### W2. 홈(대시보드) 라우팅 불명확
- **현상**: 루트 URL(`/`)이 로그인된 FC 유저의 경우 대시보드를 보여줘야 하나 `/candidates/`로 리다이렉트
- **영향**: 사이드바의 "홈" 링크를 클릭해도 후보자 검색 페이지가 나타남. FC 대시보드(할 일, 미팅 일정)에 접근할 수 없음
- **추정 원인**: `accounts/views.py`의 `home()` 함수가 로그인 안 된 상태에서는 후보자 페이지로 redirect 하는 것으로 보임

#### W3. HTMX 스냅샷 참조 빠른 무효화
- **현상**: HTMX `hx-trigger` 기반 자동 요청(예: `keyup changed delay:300ms`)이 비동기로 DOM을 변경하면서 인터랙션 요소 참조가 급격히 무효화
- **영향**: 검색 입력 중 자동 필터링이 의도치 않게 다른 페이지 컨텐츠를 로드할 수 있음

#### W4. Tailwind CSS CDN 사용 (프로덕션 경고)
- **현상**: `cdn.tailwindcss.com`을 사용 중이며, 매 페이지 로드마다 콘솔에 경고 출력
- **영향**: 프로덕션 환경에서 성능 저하 및 CDN 의존성 위험. 407KB의 Tailwind JS를 매번 다운로드

#### W5. 로딩 인디케이터 부재
- **현상**: HTMX 요청 중 `.htmx-indicator` 또는 스켈레톤 로딩 UI가 없음
- **영향**: 네트워크 지연 시 사용자에게 피드백 없이 무반응 상태로 보임. 특히 AI 브리핑, 임포트 등 오래 걸리는 작업에서 문제

### 🟢 Minor (개선 권장)

#### M1. 후보자 검색 페이지에 사이드바/하단 네비 없음
- **현상**: `/candidates/` 페이지는 별도 레이아웃을 사용하여 사이드바와 하단 네비게이션이 없음
- **영향**: CRM 영역과의 네비게이션 단절. 후보자 검색에서 연락처로 이동하려면 브라우저 뒤로가기 또는 직접 URL 입력 필요

#### M2. 폼 유효성 검사 피드백 부족
- **현상**: 필수 필드(이름*)는 마커가 있으나, 빈 폼 제출 시 HTML5 기본 유효성 검사만 동작. 커스텀 에러 메시지 없음
- **권장**: 인라인 한국어 에러 메시지 + 에러 필드 포커스

#### M3. 전화번호 링크가 `tel:` 프로토콜 미사용
- **현상**: 연락처 상세의 전화번호가 링크로 되어 있으나 `tel:` 프로토콜 사용 여부 미확인
- **권장**: 모바일에서 전화번호 클릭 시 바로 전화 앱이 열려야 함

#### M4. 미팅 생성 폼의 날짜 형식이 영문
- **현상**: 날짜 입력 필드 플레이스홀더가 `mm/dd/yyyy`로 영문 표시
- **권장**: 한국어 환경에 맞게 `yyyy-mm-dd` 또는 한글 플레이스홀더 사용

## 보이스 채팅 UX 상세 리뷰

### 구현 상태
- **텍스트 입력**: 정상 동작. 입력 후 전송 버튼 또는 Enter 가능
- **음성 녹음**: MediaRecorder API 사용, webm 형식으로 서버 전송
- **상태 전환**: idle(마이크 아이콘) → recording(빨간색) → processing(스피너) → idle 순환
- **에러 처리**: 마이크 권한 거부 시 한국어 토스트 메시지
- **채팅 히스토리**: sessionStorage 기반 세션 ID로 이전 대화 로드
- **결과 연동**: 검색 결과를 후보자 리스트에 반영 (htmx.ajax)

### 장점
- XSS 방지를 위한 escapeHtml 함수 사용
- 세션 기반 대화 히스토리 유지
- 검색 후 URL에 session_id 반영 (공유 가능)
- 모달 외부 클릭(overlay) 및 ESC로 닫기 가능

### 개선 필요
- 음성 검색 시 "듣고 있습니다" 등의 시각적 피드백 추가 필요
- 긴 응답 시 typing indicator 외에 프로그레스 바 고려
- 음성 입력 최대 시간 제한 없음 (무한 녹음 가능)
- 오프라인 상태에서 음성 검색 시도 시 에러 처리 부족

## 개선 권장사항 (우선순위)

### P0 — 즉시 수정 (배포 차단)
1. **HTMX 네비게이션 아키텍처 통합** (C1): 후보자 검색과 CRM 페이지가 동일한 base template과 HTMX 타겟 구조를 사용하도록 통합, 또는 후보자 페이지를 완전히 별도 앱으로 분리
2. **인증 보호 일관성** (C2): `/contacts/`, `/meetings/`, `/contacts/import/` 등 CRM 페이지에 `@login_required` 데코레이터 적용
3. **hx-include 셀렉터 수정** (C3): `[name='industry']` 숨겨진 input 추가 또는 hx-include 속성 제거

4. **편집 폼 드롭다운 pre-select** (W1): 기존 데이터로 초기값 설정
5. **HTMX 로딩 인디케이터 추가** (W5): `.htmx-indicator` 클래스로 스피너/스켈레톤 표시
6. **홈 라우팅 수정** (W2): 비로그인 사용자의 루트 URL 동작 명확화

### P2 — 다음 스프린트
7. Tailwind CSS 빌드 타임 컴파일로 전환 (W4)
8. 폼 인라인 유효성 검사 한국어 메시지 (M2)
9. 미팅 날짜 형식 한국어화 (M4)
10. 후보자 검색 ↔ CRM 간 네비게이션 연결 (M1)

## 배포 가능 여부 판단

### 결론: **조건부 배포 가능**

P0 항목 3개를 수정하면 배포 가능. 특히 C1(HTMX 네비게이션 충돌)과 C2(인증 부재)는 반드시 해결해야 함.

**즉시 배포 가능한 영역:**
- 후보자 검색 페이지 (`/candidates/`) — 독립적으로 동작, UI 완성도 높음
- 로그인/온보딩 플로우 — 카카오 로그인 + 역할 선택
