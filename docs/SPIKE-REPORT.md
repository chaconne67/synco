## Spike 1: Android 통화 녹음

### 핵심 발견
- Android 12+ 에서 서드파티 앱의 직접 통화 녹음은 **완전 차단**
  - `AudioSource.VOICE_CALL` → SecurityException
  - Accessibility Service 방식 → Google Play 정책 금지 + OS 차단
  - `MIC` 소스 → 자기 목소리만 녹음됨
- **삼성 내장 통화 녹음이 유일한 현실적 경로**
  - 한국 삼성 폰: 통화 녹음 기본 활성화 (한국법 일방 동의 허용)
  - 녹음 파일 경로: `/storage/emulated/0/Recordings/Call/`
  - 파일 형식: `.m4a` (AAC) 또는 `.amr`
  - 파일명: `Call recording [연락처명] [날짜] [시간].m4a`

### MVP 구현 방식
```
통화 종료 감지 (TelephonyCallback)
    ↓
삼성 녹음 폴더 모니터링 (FileObserver)
    ↓
새 녹음 파일 → CallLog 타임스탬프로 매칭
    ↓
서버 전송 → STT (Whisper/Clova) → AI 요약
    ↓
CRM에 통화 요약 자동 기록
```

### 필요 권한
- `READ_PHONE_STATE` — 통화 시작/종료 감지
- `READ_CALL_LOG` — 발신자 번호, 시간, 통화시간
- `READ_MEDIA_AUDIO` (Android 13+) — 녹음 파일 접근
- `FOREGROUND_SERVICE` — 백그라운드 모니터링

### 배포 제약
- **Google Play: 리스크 높음** — `READ_CALL_LOG`는 기본 전화/SMS 앱만 허용
- **대안: 사이드로딩 (APK 직접 배포)** 또는 **원스토어 (삼성 한국 기기 기본 설치)**
- ACR, Cube ACR 등 기존 통화녹음 앱도 동일 방식 (Play Store 퇴출 → APK 배포)

### 법적 사항 (한국)
- **통신비밀보호법**: 한국은 **일방 동의** 허용 → 녹음 자체는 합법
- **개인정보보호법**: 녹음 파일 서버 전송 시 사용자 동의 필요, 개인정보 처리방침 필수
- STT 서비스 이용 시 데이터 처리 계약 확인 (Clova: 한국 서버, Whisper: 해외)

## Spike 2: iOS 통화 캡처

### 핵심 발견
- **iOS에서 통화 녹음은 불가능.** 어떤 API로도 안 됨.
- iOS 18.1에서 Apple이 자체 통화 녹음 추가했지만 **서드파티 API 없음**
- 머지콜 방식 (TapeACall 등): 기술적으로 가능하지만 UX 나쁨 + 비용 발생

### 활용 가능한 것
| 기능 | API | 용도 |
|------|-----|------|
| 발신자 ID 표시 | CallDirectory Extension | CEO DB에서 이름+메모 표시 |
| 통화 종료 감지 | CXCallObserver | 통화 후 메모 알림 트리거 |
| 음성 메모 녹음 | AVAudioRecorder | 1-tap 통화 요약 녹음 |
| 홈 화면 위젯 | WidgetKit | "통화 메모" 빠른 접근 |

### MVP 구현 방식 (iOS)
```
통화 종료 감지 (CXCallObserver)
    ↓
로컬 알림: "통화 메모를 남기세요"
    ↓
1-tap 음성 메모 녹음 (30-60초)
    ↓
AI 요약 (Whisper STT → GPT 구조화)
    ↓
CRM 자동 기록 (연락처명, 논의 내용, 다음 액션)
```

### 주의사항
- CXCallObserver는 **전화번호를 제공하지 않음** → 사용자가 직접 선택하거나, CallDirectory 데이터로 추론
- iOS 비중 ~30-35% (FC 대상)
- **음성 메모 방식은 FC 습관 형성이 관건** → AI 프롬프트로 유도: "누구와? 어떤 내용? 다음 할 일?"

## Spike 3: 카카오톡 채팅 파싱

### 핵심 발견
- **카카오톡 공개 API: 채팅 접근 불가.** 영원히 없을 가능성 높음
- **채팅 내보내기 txt 파싱이 가장 현실적**
  - 사용자가 수동으로 내보내기 → 앱에 공유/업로드
  - 정형화된 포맷이라 파싱 안정적
- **Android NotificationListenerService**: 실시간 알림 캡처 가능 (Phase 2)

### 내보내기 파일 포맷
```
김철수 님과 카카오톡 대화
저장한 날짜 : 2025-06-15 14:23

2025년 5월 1일 목요일

[김철수] [오후 2:30] 안녕하세요, 다음 주 미팅 가능하신가요?
[나] [오후 2:31] 네, 화요일 오전 10시 어떠세요?
```

### 추출 가능한 CRM 데이터
| 데이터 | 추출 방법 | 가치 |
|--------|-----------|------|
| 상대방 이름 | 헤더 파싱 | 연락처 매칭 |
| 메시지 시간 | `[오전/오후 H:MM]` 정규식 | 접촉 빈도, 최근성 |
| 전화번호 | `01X-XXXX-XXXX` 정규식 | 연락처 자동 추가 |
| 미팅 언급 | 키워드: 미팅, 약속, 만나, 방문 | 일정 자동 등록 |
| 날짜/시간 표현 | "다음 주 화요일", "5월 3일" | 일정 추출 |
| 금액 표현 | `\d+만원`, `\d+억` | 거래 규모 추정 |
| 메시지 방향 | `[나]` vs `[상대방]` | 관계 주도권 분석 |
| 대화 공백 | 날짜 간격 계산 | 미연락 알림 |

### MVP 구현 방식
```
카카오톡 → 대화 내보내기 → txt 파일 생성
    ↓
synco 앱으로 공유 (Android Share Intent / 파일 업로드)
    ↓
서버에서 파싱 (정규식 + NLP)
    ↓
구조화된 데이터만 저장 (원본 텍스트 삭제)
    ↓
CRM 자동 업데이트: 마지막 연락일, 키워드, 일정
```

### 법적 주의사항
- 상대방 메시지도 포함됨 → 상대방 동의 없이 서버 저장은 **법적 회색지대**
- **완화 전략**: 원본 텍스트 서버 미저장, 메타데이터(빈도/키워드)만 추출, 온디바이스 처리 권장
- 개인정보보호법 전문가 자문 필요 (출시 전)

## MVP 결론: 플랫폼별 데이터 수집 전략

### Android (FC ~70%)
| 데이터 소스 | 방식 | 자동화 수준 | Phase |
|-------------|------|-------------|-------|
| 통화 내역 | CallLog API | 완전 자동 | 1 |
| 통화 녹음 | 삼성 내장 녹음 파일 읽기 | 반자동 (초기 설정 후 자동) | 1 |
| 카카오톡 | 채팅 내보내기 업로드 | 수동 (2-3탭) | 1 |
| 카카오톡 알림 | NotificationListener | 자동 (실시간) | 2 |
| 수동 메모 | 음성 입력 | 수동 | 1 |

### iOS (FC ~30%)
| 데이터 소스 | 방식 | 자동화 수준 | Phase |
|-------------|------|-------------|-------|
| 통화 감지 | CXCallObserver | 자동 (번호 없음) | 1 |
| 발신자 표시 | CallDirectory Extension | 자동 | 1 |
| 통화 요약 | 1-tap 음성 메모 → AI | 반수동 | 1 |
| 카카오톡 | 채팅 내보내기 업로드 | 수동 | 1 |
| 수동 메모 | 음성 입력 | 수동 | 1 |

### Phase 1 MVP의 현실적 위치
- **"제로 노력"은 불가능.** Android에서도 완전 자동은 통화 녹음 파일 읽기 정도
- **"최소 노력"은 가능.** 통화 → 자동 감지+요약 (Android) / 1-tap 메모 (iOS) + 카톡 내보내기 가끔 업로드
- 기존 CRM 대비 압도적으로 적은 노력 → FC 채택 가능성 있음

## 배포 전략 결정

### PWA vs 네이티브 앱

| | PWA (웹앱) | 네이티브 앱 |
|---|---|---|
| 통화 녹음 접근 | **불가** | 가능 (Android) |
| CallLog 접근 | **불가** | 가능 |
| 알림 모니터링 | **불가** | 가능 |
| 파일 업로드 | 가능 | 가능 |
| 설치 허들 | 낮음 | 중간 |

**결론: 핵심 기능(통화 연동)이 PWA에서 불가능. 네이티브 앱이 필수.**

### 권장 배포
- **Phase 1 MVP**: 웹앱 (FastAPI + HTMX) = 대시보드/CRM 관리 화면
- **Phase 1 MVP**: Android 앱 (Kotlin) = 통화 감지 + 녹음 파일 접근 + 백그라운드 서비스
- **Phase 2**: iOS 앱 추가 (Swift) = 통화 감지 + 음성 메모
- 웹앱과 네이티브 앱은 같은 FastAPI 백엔드를 공유
