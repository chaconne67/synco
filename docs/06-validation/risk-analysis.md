## High-Priority Risks 상세

### R1: CEO WTP 부재 🔴

**시나리오:** CEO가 매칭 정보에 돈을 낼 의향이 없다. 무료 정보는 보지만 크레딧 구매를 하지 않는다.

**근거 (Wave 3 리서치 업데이트):**
- CEO WTP 직접 데이터 없음이나 간접 벤치마크 강화됨 [Data]:
  - **리멤버 Black**: 연 30만원 (250만+ 회원, 매출 684억, EQT 약 5,400억원 인수) — B2B 네트워킹 과금 검증
  - **BNI Korea**: 연 약 135~405만원 (~3,000명 회원) — 대면 비즈 매칭 높은 WTP
  - **골프 회원권**: 3~5억원 — CEO 네트워킹 수단으로 극도의 WTP
- 한국 B2B SaaS 프리미엄 전환율: 2~5%, PLG 최대 ~9% [Data, 2025]
- CEO 평균 55세, 70% 50대 이상, 도소매/제조업 중심 [Data] — 디지털 과금 저항 가능성

**조기 경보:**
- Exp 1(인터뷰)에서 "돈 낼 의향" 60% 미만
- Exp 4(컨시어지 MVP)에서 블러 해제 결제 3건 미만
- Month 5까지 유료 크레딧 구매 CEO 3명 미만

**완화 전략:**
1. 컨시어지 MVP로 실제 결제 행동 조기 검증 (Exp 4)
2. 가격 A/B 테스트로 최적점 탐색 (Exp 5)
3. 실패 시 대안: 매칭 무료 + 거래 성사 수수료 모델로 피봇

### R2: GA 파트너십 실패 / FC 활용도 저조 🟡 (기존 🔴에서 하향)

**시나리오:** GA 법인이 파트너십에 관심이 없거나, 파트너십 체결 후에도 FC의 실제 도구 활용도가 저조하다.

**근거:**
- GA는 보수적 조직일 수 있음 — 신규 플랫폼에 대한 관망 자세
- FC는 자영업자이므로 GA 권유에도 강제력 한계
- DB 가공 서비스의 체감 가치가 기대에 못 미칠 수 있음

**리스크 하향 사유 (🔴→🟡):**
- "FC가 CEO를 데려온다"는 가정이 아님. DB 소유 주체는 GA(법인)이고, FC는 GA와 계약 관계의 자영업자
- GA를 지분/수익 쉐어로 파트너로 끌어들이면 FC는 GA의 지시/권유로 자연 편입
- 공동창업자가 이미 2개 GA 계약 보유 → Day 1 실행 가능
- GA 입장: 이미 돈 주고 사는 DB를 synco가 AI로 가공 + 신규 수익원 = 거절 이유 적음

**조기 경보:**
- GA 파트너십 제안에 3개 GA 연속 거절
- 파트너 GA에서 FC 앱 사용률 30% 미만 (Month 2)
- DB 가공 브리핑에 대한 FC 만족도 3.0 미만

**완화 전략:**
1. 공동창업자 GA부터 성공 사례 확보 → 레퍼런스로 다음 GA 확장
2. DB 가공 품질 확보: AI 브리핑이 FC 방문 성과에 직결되는 것을 입증
3. GA 대표에게 지분 참여 옵션 제시 — 단순 계약이 아닌 공동 성장 구조
4. 실패 시 대안: CEO 직접 마케팅 (SEO, 리멤버 광고) — 비용 발생

### R3: 매칭 품질 부족 🟡

**시나리오:** CEO가 가입하고 프로필을 작성했지만 AI 매칭 결과가 "뻔한" 또는 "안 맞는" 기회만 보여준다.

**근거:**
- 초기 CEO 수 적음 → 매칭 후보 풀 부족
- CEO 프로필 데이터 품질 불균일 (업종/자원 기술 수준 차이)
- AI 매칭 알고리즘 자체의 한계

**조기 경보:**
- CEO 매칭 피드 열람 후 재방문율 30% 미만
- 블러 해제 후 "이건 아니다" 피드백 50% 초과
- NPS -10 이하

**완화 전략:**
1. 초기 50명까지 수동 큐레이션 보완
2. 프로필 완성도 80%+ 보너스 크레딧으로 데이터 품질 확보
3. "매칭 없음"보다 "아직 데이터 쌓이는 중, n주 후 재확인" 메시지
4. 매칭 범위를 "유휴 자원" 외 "공동 마케팅", "인맥 교환"까지 확장

---

### R4: CEO 프로필 미완성 🟡

**시나리오:** CEO가 가입은 하지만 프로필을 대충 작성하거나 핵심 정보(자원, 관심 분야)를 비워둔다. 매칭 품질 전체가 무너진다.

**근거:**
- 정보 입력은 "귀찮은 작업" — 즉각적 보상 없으면 동기 부족
- CEO가 자기 사업 정보를 공개하는 것에 대한 보안 우려
- User Journey 분석에서 "최대 드롭오프 지점"으로 식별됨

**조기 경보:**
- 프로필 완성율(80%+) 15% 미만
- 평균 입력 시간 2분 미만 (대충 넘기는 신호)
- "자원" 필드 공란율 50% 초과

**완화 전략:**
1. 보너스 크레딧 5cr (2.5만원 상당) — 프로필 80%+ 시 지급
2. 단계적 입력: 핵심 3개 필드만 필수, 나머지 점진적 요청
3. "입력하면 이런 매칭이 나옵니다" 예시 미리 보여주기
4. FC가 대면에서 함께 작성 유도 (FC 대면 소개 시나리오에 포함)

---

### R5: PIPA 2026 개인정보 리스크 🔴 (기존 🟡에서 상향)

**시나리오:** FC가 보유한 CEO 고객 정보를 플랫폼에 옮기는 과정에서 개인정보보호법 위반 이슈 발생. PIPA 2026 하에서 과징금이 **전 세계 매출 10%**까지 가능.

**근거 (Wave 4 리서치 업데이트):**
- PIPA 2026 개정안 (2026.2 공포, **2026.9.11 시행**) [Data, 공식 법령]:
  - 과징금 상한 매출 3% → **10%** (1,000만명+ 정보주체 중대 위반 시)
  - **CEO 감독 책임 법제화** — 경영진 개인 책임
  - 대규모 개인정보 처리자 **ISMS-P 인증 의무화**
- FC가 보유한 CEO 연락처는 보험 목적 수집 → 목적 외 사용 시 **명시적·구체적 동의** 필수 [Data]
- CEO가 직접 가입하는 구조이므로 위험은 낮지만, FC가 기존 DB를 업로드하는 경우 문제 발생 가능

**긍정적 요소:**
- **B2B 매칭 플랫폼 별도 허가 불요** — 일반 사업자등록 + PIPA 준수로 운영 가능 [Data]
- 보험/GA 맥락의 B2B 접촉 데이터 플랫폼 관련 **집행 선례 없음** (2024-2026) [Data]
- 보험업법상 FC 부업 직접 금지 조항 없음 (이해충돌·이중등록 규정만 적용) [Data]

**완화 전략:**
1. Phase 1 초기에 법률 자문 200~500만원 투자 (유일한 필수 초기 비용)
2. FC의 엑셀 임포트 시 "이 고객에게 가입 초대를 보냅니다" → CEO 직접 가입 → 자기 정보 직접 입력 구조
3. FC DB 직접 업로드가 아닌 "초대 링크 전송" 방식으로 위험 최소화
4. PIPA 2026 시행(2026.9) 전까지 컴플라이언스 체계 완비
