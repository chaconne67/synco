# synco 사업계획서

## 1. 사업 개요

### 한 줄 요약
AI 네이티브 헤드헌팅 플랫폼. 중소 서치펌에 맞춤형 웹 어플리케이션을 제공하고, 후보자 DB 공유 네트워크를 구축하여 업계 전체의 매칭 효율을 높인다.

### 비전
한국 헤드헌팅 업계의 "한방(부동산 매물 공유 시스템)"이 된다.

### 핵심 가치
- 후보자 DB는 공유할수록 가치가 올라간다
- AI가 이력서를 읽고, 검증하고, 매칭한다
- 컨설턴트는 사람을 만나는 일에 집중한다

## 2. 시장 분석

### 국내 헤드헌팅 시장

| 항목 | 수치 |
|------|------|
| 시장 규모 | ~1.2조원 (2025) |
| 서치펌 수 | ~2,000~4,000개 |
| 헤드헌터 수 | ~6,000~8,000명 |
| 1위 서치펌 매출 | 200억+ |
| 대다수 서치펌 | 5~20인 소규모, 엑셀+카톡 운영 |
| 1인당 평균 매출 | ~4,000만원 |

### 시장의 문제

1. **DB 사일로**: 각 서치펌이 자체 DB만 보유. 규모가 작을수록 후보자 풀이 빈약
2. **이력서 관리 부재**: 워드 파일을 폴더에 쌓아둠. 검색 불가, 최신화 안 됨
3. **수작업 과다**: 이력서 읽기, 후보자 분류, 서류 작성 모두 수작업
4. **위조 탐지 불가**: 복수 이력서 간 불일치를 사람이 일일이 비교
5. **중복 컨택**: 같은 후보자에게 여러 컨설턴트가 동시 접근

### 경쟁 환경

#### 해외 경쟁사

| 경쟁사 | 펀딩 | 매출 | 한계 |
|--------|------|------|------|
| Loxo (미국) | $115M | $112M | 한국어 미지원, 비쌈 |
| Recruiterflow (미국) | 부트스트랩 | $24M | 한국어 미지원 |
| Recruit CRM (인도) | 부트스트랩 | 비공개 | 한국 이력서 파싱 불가 |
| Manatal (태국) | $5.1M (Sequoia) | $6M | 한국 특화 기능 없음 |

해외 제품은 한국어 이력서 파싱 불가, 한국 자격증/대학 체계 미지원, 영어 전용 UI, 한국 결제 불편 등으로 국내 서치펌이 실제 사용하기 어렵다.

#### 국내 경쟁사

**A. 서치펌 내부 도구 (직접 경쟁)**

| 경쟁사 | 성격 | 강점 | 한계 |
|--------|------|------|------|
| 유아이파트너스 | 서치펌 DB관리 프로그램. 10년 운영, 인재 DB+매칭+리포트 올인원 | 업계 신뢰도, 검증된 운영 이력 | AI 기능 없음, 전통적 SI 납품 형태 |
| 대형 서치펌 자체 시스템 | 프로매치코리아(30만건 DB), 커리어앤스카우트 등 | 방대한 DB, 내부 최적화 | 자사 전용, 외부 판매 안 함 |

유아이파트너스가 유일한 직접 경쟁자이나, AI 이력서 파싱/자연어 검색/위조 탐지 등 AI 네이티브 기능이 없는 전통적 DB 관리 프로그램이다.

**B. 채용 중개 플랫폼 (간접 경쟁, 다른 카테고리)**

| 플랫폼 | 성격 | synco와의 차이 |
|--------|------|---------------|
| 히든스카우트 | 기업↔서치펌 연결 플랫폼. 3,200명 헤드헌터 네트워크 | 기업 대상 서비스. 서치펌이 쓰는 내부 도구가 아님 |
| 리멤버 | 명함 앱에서 경력직 채용 플랫폼으로 확장. AI+빅데이터 | 리멤버 자체가 헤드헌팅 수행. 서치펌을 위한 도구가 아님 |
| 잡인덱스 | 서치펌/헤드헌터 정보 포털 | 정보 사이트, 업무 도구 아님 |

이 플랫폼들은 "기업이 서치펌을 찾는" 또는 "플랫폼 자체가 헤드헌팅하는" 서비스로, 서치펌 컨설턴트가 일상 업무에 사용하는 도구가 아니다.

#### synco vs 유일한 국내 경쟁자 비교

| 기능 | 유아이파트너스 | synco |
|------|--------------|-------|
| 인재 DB 관리 | O | O |
| AI 이력서 자동 파싱 | X | **O** |
| AI 자연어 검색 | X | **O** |
| 보이스 입력 | X | **O** |
| 이력서 위조/변조 탐지 | X | **O** |
| DB 공유 네트워크 | X | **O** |
| 중복 컨택 방지 | 불명 | **O (예정)** |
| 서류 자동 작성 | 불명 | **O (예정)** |
| 업계 운영 실적 | **O (10년)** | X (신규) |
| 기존 고객 기반 | **O** | X (구축 중) |

**결론: 국내에 AI 네이티브 헤드헌팅 전문 플랫폼은 없다.** 유아이파트너스가 유일한 경쟁자이나 AI 세대 차이가 있으며, 해외 제품은 한국 시장에 진입하지 못한 상태다.

### synco 차별점 (경쟁사가 못 하는 것)

1. **이력서 위조/변조 탐지** — 동일인 복수 이력서 비교, 시간 모순 탐지, AI 2차 분석. 국내외 어떤 제품에도 없는 기능
2. **보이스 퍼스트** — 음성으로 후보자 검색, 상담 기록, 서류 작성
3. **한국 헤드헌팅 특화** — 한국어 이력서 파싱, 인서울 대학 분류, 한국 자격증 체계
4. **DB 공유 네트워크** — 부동산 매물 공유 모델 적용, 수수료 분배 구조
5. **컨설턴트 간 중복 컨택 방지** — Job Order 기반 컨택 이력 공유

## 3. 서비스 구조

### 3.1 핵심 기능

#### A. 이력서 자동 처리 파이프라인
```
이력서 파일 도착 (이메일/드라이브)
→ 텍스트 추출 (doc/docx/pdf/hwp)
→ AI 구조화 추출 (이름, 경력, 학력, 자격증, 연봉 등)
→ 신뢰도 평가 (원본 확인/AI 추론/AI 생성 구분)
→ DB 저장 + 자동 분류 (직무 카테고리)
→ 파일명 자동 생성 (이름.YY.회사1.회사2.대학교)
```

#### B. AI 후보자 검색
```
자연어 입력 (텍스트/음성)
"인서울 출신 AICPA 보유 남성, 경력 10년 이상"
→ AI가 SQL 자동 생성
→ DB 검색 (candidates + careers + educations + certifications)
→ 결과 표시 + 대화형 조건 좁히기
```

#### C. 이력서 Discrepancy 탐지
```
3계층 탐지:
Layer 1: 버전 간 비교 — 과거 사실이 사라졌는가
Layer 2: 내부 일관성 — 시간 겹침, 학력-나이 모순
Layer 3: 이상 패턴 — 비정상적 직무 범위, 학력 구멍

출력: RED(위조 의심) / YELLOW(확인 필요) / BLUE(참고)
```

#### D. Job Order 관리 + 중복 컨택 방지
```
Job Order 등록 (고객사 + 포지션)
→ 후보자 추출 (AI 검색)
→ 컨택 기록 (일시, 방법, 내용)
→ 다른 컨설턴트가 같은 후보자 검색 시 경고
→ 라이프사이클: 신규 → 진행 → 추출 → 서류제출 → 면접 → 클로즈
```

#### E. 서류 자동 작성
```
이력서 DB + 컨설턴트 상담 메모 (음성/텍스트)
→ AI가 회사 양식(CV_E, CV_K)에 맞게 서류 생성
→ 개인정보/연봉 정보 자동 삭제 (고객사 제출용)
→ 오탈자, 문법, 폰트, 서식 자동 통일
→ 컨설턴트 확인 후 PDF/Word 출력
```

#### F. 이력서 자동 업데이트 요청
```
1년 이상 된 이력서 자동 감지
→ 후보자 이메일로 업데이트 요청 자동 발송
→ 회신된 이력서 자동 파싱 + DB 업데이트
```

#### G. 업계 뉴스 큐레이션
```
설정된 소스 (뉴스, 유튜브, SNS) 모니터링
→ 채용/인사/HR 관련 기사 필터링
→ 일일 브리핑 발송 (이메일/앱)
```

#### H. 수익 정산
```
성사 건별 수수료 기록
→ 컨설턴트별 인센티브 자동 계산
→ DB 공유 수수료 분배 자동 정산
→ 간이 회계 리포트
```

### 3.2 DB 공유 네트워크

부동산 "한방" 모델 적용:

```
[DB 오너 서치펌]                    [컨택 서치펌]
   이력서 제공 ──→ synco 공유 DB ←── 검색/컨택
                      ↓
                후보자 매칭 성사
                      ↓
              수수료 (연봉의 20~25%)
                      ↓
           DB 오너 20~30% : 컨택측 70~80%
```

**참여 조건**: synco 사용 + 자기 DB 공유 동의
**접근 제어**: 전체 DB 노출이 아닌, 검색 결과만 노출. 연락처는 DB 오너 중개

## 5. 초기 진입 전략 (Go-to-Market)

### 앵커 클라이언트 전략

```
Step 1: 기존 클라이언트 (30년 경력, 20만건 DB) synco 완성
Step 2: 기존 클라이언트가 영업 파트너로서 지인 서치펌에 직접 제안
        "내 20만건 DB 쓸 수 있어, 500만원이면 돼"
Step 3: 5곳 온보딩 → 각 3,000~5,000건 추가 → 22만건
Step 4: 성사 사례 1~2건 만들기 (DB 공유로 매칭 성공)
Step 5: 성공 사례로 다음 10곳 영업
Step 6: 15곳 달성 → 25만건 네트워크 → 본격 확산
```

### 왜 작동하는가

- 20만건 DB = 중소 서치펌 보유량의 20~100배 → 거절하기 어려운 제안
- 500만원 = 성사 1건이면 회수 → 리스크 최소
- 기존 클라이언트의 업계 신뢰도 (30년) = 영업 자체가 추천
- 부동산 매물 공유와 동일 구조 → 업계에 설명하기 쉬움

## 6. 기술 스택 및 현황

### 개발 완료 (현재)
- 후보자 DB + AI 검색 (자연어 → SQL)
- 이력서 파싱 파이프라인 (Google Drive → 텍스트 추출 → LLM 구조화)
- 추출 신뢰도 표시 (원본/AI 추론/AI 생성 구분)

- 카카오 로그인
- 반응형 웹 UI (모바일/데스크톱)

### 개발 진행 중
- 이력서 Discrepancy 탐지
- 누락 필드 보완 추출 (연봉, 병역, 수상 등)
- 중복 이력서 머지

### 개발 예정
- Job Order 관리 + 라이프사이클
- 컨설턴트 간 중복 컨택 방지
- 서류 자동 작성 (CV_E, CV_K 양식)
- 이력서 자동 업데이트 요청 이메일
- 업계 뉴스 큐레이션
- 수익 정산 시스템
- DB 공유 네트워크 (접근 제어, 수수료 분배)

### 기술 스택
- Backend: Django 5.2 + PostgreSQL (pgvector)
- Frontend: HTMX + Tailwind CSS
- AI: Claude API (이력서 파싱, 검색, 분석)
- 음성: OpenAI Whisper
- 배포: Docker Swarm + Nginx

## 7. 해외 진출 전망

### 우선순위

| 시장 | 가능성 | 시기 | 전제 조건 |
|------|--------|------|----------|
| 일본 | 6/10 | 3~5년차 | 국내 DB 네트워크 증명 + 일본어 파싱 |
| 동남아 | 5/10 | 3~5년차 | 영어 지원 + 현지 파트너 |
| 미국/유럽 | 2/10 | - | 비추천 (Loxo 등 강자 포진) |

### 해외 진출의 열쇠

DB 공유 네트워크가 국내에서 증명되어야 한다. 증명 없이 나가면 "그냥 또 하나의 리크루팅 SaaS"로 Loxo와 싸워야 하고, 그건 이길 수 없다.

DB 네트워크가 증명되면 각 나라 현지 서치펌을 온보딩하면서 해당 국가 후보자 DB가 자동으로 쌓이는 플라이휠이 작동한다. 이것은 LinkedIn 크롤링 기반인 Loxo가 가지지 못한 구조.

## 8. 경쟁 방어 전략 — "대기업이 따라하면?"

### 8.1 따라할 수 있는 것 vs 없는 것

| 따라할 수 있는 것 (기술) | 따라할 수 없는 것 (네트워크) |
|---|---|
| AI 이력서 파싱 | 20만건 시드 DB + 앵커 파트너 |
| 자연어 검색 | 서치펌 네트워크의 신뢰 관계 |
| 보이스 입력 | 30년 업계 인맥이 추천하는 영업 구조 |
| 위조 탐지 알고리즘 | 이미 쌓인 DB 공유 네트워크 효과 |

기술은 6개월이면 따라 만든다. DB 네트워크는 6년이 걸린다.

### 8.2 잠재 경쟁자별 진입 장벽

| 잠재 경쟁자 | 기술력 | 진입 가능성 | 구조적 장벽 |
|------------|--------|-----------|-----------|
| 리멤버 | 높음 | **낮음** | 이미 "플랫폼이 직접 헤드헌팅"하는 모델. synco 모델(서치펌을 강하게 만드는 도구)을 따라하면 자기 비즈니스와 충돌 |
| 사람인/잡코리아 | 중간 | **낮음** | 채용 공고 플랫폼. 서치펌 도구를 만들면 자기 고객(기업 HR)에게 "헤드헌터 쓰세요"라고 권하는 셈 |
| 대형 SI | 높음 | **낮음** | 최소 프로젝트 단가 수억원. 500만원짜리 중소 서치펌 납품은 비즈니스가 안 됨 |
| 유아이파트너스 | 중간 | **중간** | AI 기능 추가 가능하나 10년간 전통 SI 모델의 코드베이스+조직 전환 필요. 앵커 DB 20만건 없음 |
| 해외 SaaS (Loxo 등) | 높음 | **낮음** | 한국어 이력서 파싱 + 한국 자격증/대학 체계 + 한국 서치펌 네트워크를 처음부터 구축해야 함 |

### 8.3 핵심 방어 논리

synco의 해자(moat)는 기술이 아니라 네트워크다.

부동산 "한방"을 네이버가 따라 만들 수 있었지만 안 했다. 공인중개사 협회의 신뢰 관계와 매물 네트워크를 기술로 대체할 수 없었기 때문이다.

synco도 마찬가지다. 30년 경력 서치펌 대표가 자기 20만건 DB를 걸고 직접 영업 파트너로 나서는 구조다. 대기업이 기술을 따라 만들 수는 있지만, 이 사람의 전화 한 통을 대체할 수는 없다.

네트워크는 시간이 갈수록 강해진다. 서치펌 10개가 모이면 20개가 쉽고, 20개가 모이면 50개가 쉽다. 먼저 시작한 쪽이 이긴다.

### 8.4 방어 타임라인

| 시기 | 방어 수단 | 후발 주자 진입 난이도 |
|------|----------|-------------------|
| 지금~1년 | 기술 차별화 (AI 파싱, 검색, 위조 탐지) | 낮음 (6~12개월이면 따라 만듬) |
| 1~2년 | DB 네트워크 10~20개 서치펌 확보 | 중간 (DB를 어디서 구하나) |
| 2~3년 | DB 25만건+ 네트워크 효과 발동 | 높음 (빈 DB로는 매력 없음) |
| 3년+ | 업계 표준 인프라 ("헤드헌팅하려면 synco") | 매우 높음 (전환 비용 + 네트워크 락인) |

첫 2년이 경쟁의 전부다. 2년 안에 네트워크를 만들면 대기업이 와도 늦다.

| 클라이언트 이탈 | 중간 | DB가 쌓일수록 전환 비용 증가 (lock-in), 유지보수 품질 관리 |

## 11. 요약

synco는 "헤드헌팅 업계의 한방"이다.

- **문제**: 중소 서치펌은 DB가 작고, 수작업이 많고, 도구가 없다
- **해결**: AI 네이티브 플랫폼 + DB 공유 네트워크
- **차별점**: 위조 탐지, 보이스 퍼스트, 한국 특화 — 해외 경쟁사에 없는 기능
- **진입 전략**: 앵커 클라이언트의 20만건 DB를 미끼로 500만원 파트너 가격 제안
- **수익**: 일시불 납품 + 유지보수 + DB 공유 수수료
- **목표**: 3년 내 50개 서치펌, 25만건 공유 DB, 연 매출 4억원
