# 이력서 데이터 추출 파이프라인 요약

## 한 줄 요약

이력서 파일에서 정보를 꺼내고, 그 정보가 앞뒤가 맞는지 자동으로 검사하는 시스템.

## 전체 흐름

### 준비 단계: 이력서 파일을 텍스트로 변환

구글 드라이브에 올려진 이력서 파일을 다운로드하고, 문서 안의 글자를 추출한다.
표, 텍스트 상자, 본문 등에서 글자를 모두 꺼낸 뒤, 중복되거나 의미 없는 줄을 정리한다.

---

### 1단계 — 있는 그대로 전부 가져오기

AI가 이력서 텍스트를 읽고, 보이는 모든 정보를 가져온다.

핵심: **골라내거나 정리하지 않는다.** 같은 회사가 국문란에도, 영문란에도,
일문란에도 나오면 세 개를 각각 따로 가져온다.
국문에는 시작일이 A년이고 일문에는 B년이면, 둘 다 그대로 가져온다.

왜 이렇게 하나: 뒷 단계에서 "국문이랑 일문이 다르네?" 하고 비교하려면,
양쪽 데이터가 다 있어야 한다. 여기서 하나를 골라버리면 비교 자체가 불가능해진다.

---

### 1.5단계 — 같은 회사끼리 묶기

1단계에서 가져온 항목들 중 같은 회사/학교를 가리키는 것들을 묶는다.

다른 언어로 쓰인 같은 회사명을 AI가 판단해서 한 묶음으로 만든다.
한 회사의 전체 재직 기간과 그 안의 세부 직무 기간도 같은 묶음.

합치지는 않는다. 누가 누구랑 같은지만 표시한다.

왜 이렇게 하나: 다음 단계에서 묶음별로 따로따로 처리하기 위해서.
각 묶음은 독립적이라 동시에 병렬로 처리할 수 있다.

---

### 2단계 — 묶음별로 정리하면서 이상한 점 발견하기

각 묶음을 AI가 하나의 깔끔한 정보로 합친다. 여러 묶음을 동시에 병렬로 처리한다.

합치려는데 국문과 일문의 날짜가 크게 다르면 — 합칠 수가 없다.
이런 **합치다가 안 되는 것**이 위조 의심 신호.
재직 기간이 2개월인데 옆에 11개월이라고 적혀 있는 것도 마찬가지 —
같은 항목 안에서 숫자가 맞지 않는다.

반면 모든 섹션의 날짜가 일치하면 매끄럽게 합쳐진다. 이러면 문제 없음.

핵심 발상: **위조를 따로 찾는 게 아니라, 정보를 합치는 과정에서 자연스럽게 드러나게 한다.**

---

### 3단계 — 회사 간 기간 겹침 확인

2단계에서 정리된 경력들을 놓고, 서로 다른 회사의 재직 기간이 겹치는지 계산한다.
AI가 아닌 코드로 처리한다 — 날짜 비교만 하면 되니까.

A회사 재직 중에 B회사도 다니고 있었다면, 겸직인지 기간 조작인지 확인이 필요하다.
다만 이직할 때 한두 달 겹치는 건 정상이고, 같은 그룹사 간 겹침도 정상.

---

### 각 단계마다 — 품질 검사 + 자동 재시도

각 단계가 끝나면 결과를 자동으로 검사한다.

- 1단계 후: "이력서에 일본어가 있는데 일문 데이터가 없다" → 피드백과 함께 재실행
- 1.5단계 후: "분류 안 된 항목이 너무 많다" → 재실행
- 2단계 후: "합친 결과에 필수 정보가 빠졌다" → 해당 묶음만 재실행

한 번 재시도해도 안 되면 "사람이 확인 필요"로 표시.

---

## 최종 출력

```
후보자 정보: 이름, 연락처, 학력, 경력 (정리된 것)
위조 의심 목록:
  - [빨강] ○○회사 시작일이 이력서의 두 섹션에서 수년 차이로 다르게 기재됨
  - [노랑] ○○회사와 ○○회사 재직기간이 상당 기간 겹침
```

빨강은 채용 담당자가 즉시 재검토해야 할 수준.
노랑은 면접 시 확인 사항으로 기록하는 수준.
의심 사항이 없으면 빈 목록.
