# t25: 통합 테스트 + 엣지 케이스

> **Phase:** 3 — 워크플로우 연결
> **선행 조건:** t19, t20, t21, t22, t23, t24 (Phase 3 전체 구현 완료)

---

## 배경

Phase 3의 개별 태스크(t19-t24)가 모두 구현된 후, 전체 워크플로우가 올바르게 동작하는지 통합 테스트를 수행하고 엣지 케이스를 검증한다. 개별 태스크의 테스트가 단위 수준이라면, 이 태스크는 경계 조건과 회귀를 확인한다.

## 요구사항

### 엣지 케이스 테스트

1. **폼 유효성 검사 실패 시 기존 동작 유지:** `submission_create()`에 필수 필드 누락 시, 폼을 다시 렌더링하고 `HX-Retarget` 헤더가 없어야 한다.

2. **중복 Submission 생성 방지:** 같은 후보자에 대해 중복 Submission 생성 시도 시, 유효성 검사 실패로 폼을 재렌더링해야 한다.

3. **유도 배너 일회성 확인:** 컨택 탭을 새로고침(탭 재로드)하면 배너가 없어야 한다. 배너는 `contact_update()` 응답에만 포함된다.

4. **퍼널 컨택 카운트에서 예정 제외:** 퍼널의 컨택 카운트에서 예정(RESERVED) 상태의 컨택은 제외해야 한다.

### 기존 테스트 회귀 확인

- Phase 1, 2에서 작성한 기존 테스트(`test_p05_project_tabs.py`, `test_p06_contacts.py` 등)가 모두 통과해야 한다.

## 제약사항

- 이 태스크는 테스트 코드만 추가하며, 프로덕션 코드를 변경하지 않는다.
- Phase 3의 모든 태스크(t19-t24)가 구현된 후에 실행한다.
