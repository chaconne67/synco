# t21: contact_update() "관심" 결과 시 추천 유도 배너

> **Phase:** 3 — 워크플로우 연결
> **선행 조건:** t19 (tabChanged 이벤트 시스템)

---

## 배경

컨택 결과를 "관심"으로 저장한 후, 다음 단계인 추천 서류 작성으로의 전환이 자연스럽지 않다. 사용자가 "관심" 결과를 입력한 직후, 추천 서류 작성을 유도하는 배너를 표시하면 워크플로우의 연속성이 개선된다.

## 요구사항

### "관심" 결과 저장 시 유도 배너 표시

`contact_update()` 뷰를 수정하여:

- 결과가 `INTERESTED`로 변경된 경우, 폼 영역에 유도 배너를 렌더링하여 반환한다.
- 배너에는 후보자 이름과 "추천 서류 작성하기" 버튼이 포함된다.
- 버튼 클릭 시 `submission_create` 폼을 `#tab-content`에 로드하고 `tabChanged` 이벤트를 발행한다.

```
┌─────────────────────────────────────────────────┐
│ ✓ 홍길동님 컨택 결과가 "관심"으로 저장되었습니다.    │
│                                                  │
│   [추천 서류 작성하기 →]                           │
└─────────────────────────────────────────────────┘
```

### 배너 표시 조건

- 결과가 `INTERESTED`이고, 해당 후보자에 대한 Submission이 아직 없는 경우에만 표시한다.
- 이미 Submission이 있으면 배너 없이 기존처럼 204를 반환한다.
- "관심" 이외의 결과로 변경한 경우에도 기존처럼 204를 반환한다.

### 배너의 일회성

- 배너는 `contact_update()` 응답에만 포함된다.
- 컨택 탭을 새로고침(탭 재로드)하면 배너가 사라진다.
- 배너는 서버 상태에 저장하지 않는다.

## 제약사항

- `contactChanged` 이벤트는 항상 발행하여 하위 호환성을 유지한다.
- 배너 HTML은 별도 파셜 템플릿(`contact_interest_banner.html`)으로 분리한다.
