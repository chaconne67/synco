# 06. UI 설계 — 3-레벨 뷰

사장님이 말씀하신 플로우 그대로. 칸반에서 프로젝트를, 프로젝트 클릭으로 후보자 리스트를, 후보자 클릭으로 상세 페이지를.

---

## 1. 전체 구조 — 3-레벨 네비게이션

```
Level 1: 프로젝트 칸반 (홈/메인 뷰)
   ↓ 카드 클릭
Level 2: 프로젝트 상세 (후보자 리스트 + 프로그레스 바)
   ↓ 행 클릭
Level 3: 후보자 상세 (M:N 관계: 이 후보자의 모든 Application)
```

---

## 2. Level 1 — 프로젝트 칸반

Phase별 컬럼으로 프로젝트 카드를 배치한 칸반.

### 레이아웃

```
┌──────────┬────────────┬──────────┬───────────┬────────┐
│ 서칭     │ 사전미팅    │ 제출/검토 │ 면접      │ 종료   │
│          │             │          │           │        │
│ [삼성전자] │ [SKBP]      │ [Vatech] │ [한미약품] │ [덕산] │
│ 10명      │ 5명         │ 3명      │ 2명       │ ✅성공 │
│ 진행 18일  │ 진행 26일    │ 진행 12일 │ 진행 45일  │        │
│          │             │          │           │        │
│ [OneStore]│ [나노신소재] │ [EWOO]   │ [ASICS]   │ [Asics]│
│ 7명       │ 8명         │ 4명      │ 1명       │ ❌실패 │
│ ...       │ ...         │ ...      │ ...       │        │
└──────────┴────────────┴──────────┴───────────┴────────┘
```

### 카드에 표시할 정보

- 프로젝트 이름 (Client | Position)
- 활성 Application 수 ("10명")
- 경과 일수 ("진행 18일")
- Phase별 추가 컨텍스트:
  - 서칭: `신규 발굴 X명`
  - 사전미팅: `미팅 기록 X건`
  - 제출/검토: `피드백 대기 X일`
  - 면접: `최근 면접 X차 진행`
  - 종료: `성공 ✅ (입사자 이름)` 또는 `실패 ❌`

### 인터랙션

- 카드 클릭 → Level 2 이동 (`/projects/<id>/`)
- 컬럼별 정렬: 경과 일수 기준 오래된 것 상단 (주의 환기)
- 드래그 앤 드롭: **비활성화**. Phase는 auto-derived이므로 수동 이동 허용하지 않음
- 필터: 컨설턴트별, 고객사별, 기간별, phase별 (상단 필터 바)

### 종료 컬럼 처리

- 기본적으로 접힌 상태 (클릭 시 펼침)
- 성공/실패 구분 표시
- 최근 N개만 보이고 "더보기" 버튼

## 3. Level 2 — 프로젝트 상세

프로젝트 카드를 클릭하면 이 프로젝트의 모든 Application을 나열.

### 상단 — 프로젝트 요약

```
┌────────────────────────────────────────────────────────┐
│ 삼성전자 | AI Engineer (대리~과장)         [ 메뉴 ▾ ]  │
│ 서칭 단계 · 진행 18일 · 10명                           │
│ JD 보기 · 컨설턴트: 박정일 · 고객사: 삼성전자          │
└────────────────────────────────────────────────────────┘
                                           [ + 후보자 추가 ]
```

### 메인 — Application 리스트 (후보자별 진행도)

각 행 = Application 1건. 프로그레스 바로 현재 stage 시각화.

```
┌──────────────────────────────────────────────────────────┐
│ 김철수  발굴       ●━━○━━○━━○━━○━━○                       │
│   1992년생 · 현대차 · 서울대 · 5년차                      │
│   노트: 관심 있음, 이력서 요청 예정                       │
│                              [ 드롭 ▼ ]  [ 다음단계 ▶ ]  │
├──────────────────────────────────────────────────────────┤
│ 이영희  사전미팅   ●━━●━━◐━━○━━○━━○                       │
│   1985년생 · 삼성SDS · KAIST · 12년차                     │
│   사전미팅: 2026-04-10 (녹음 있음)                        │
│                              [ 드롭 ▼ ]  [ 다음단계 ▶ ]  │
├──────────────────────────────────────────────────────────┤
│ 박민수  드롭됨 (경력부족)                                  │
│   1990년생 · 스타트업 · 한양대                             │
│   드롭 사유: 부적합_경력                                   │
│                                           [ 복구 ]       │
└──────────────────────────────────────────────────────────┘
```

### 프로그레스 바 디자인

5단계 원형 마커 + 연결선:

```
발굴    검토    사전미팅    추천    면접    입사
 ●━━━━━○━━━━━○━━━━━━━━○━━━━━━○━━━━━○
```

- `●` = 완료
- `◐` = 현재 진행 중
- `○` = 미진행
- 색상: 완료=진한 파랑, 현재=파랑+펄스, 미진행=회색
- 드롭 시: 현재 위치에서 X 표시 + 회색 처리

### 버튼 동작

- **다음단계 ▶**: 현재 stage의 다음 단계로 promote
  - SOURCED → SCREENED → PRE_MEETING → RECOMMENDED → CLIENT_INTERVIEWING → HIRED
  - 최종 단계(HIRED)는 별도 확인 모달 (입사일, 연봉 등 입력 옵션)
- **드롭 ▼**: 드롭 사유 선택 드롭다운 (DropReason enum)
  - 선택 후 `drop_note` 자유 텍스트 입력 옵션
- **복구 (dropped 상태에서만)**: DROPPED → 이전 stage로 되돌림

### 필터·정렬

- 현재 stage별 필터 (체크박스)
- 정렬: 최근 업데이트순 / 이름순 / stage 진행도순

### 추가 액션

- **+ 후보자 추가**: 후보자 검색/선택 모달 → 선택 시 stage=SOURCED로 Application 생성
- **프로젝트 종료**: 모든 활성 Application이 없고 HIRED도 없을 때 수동 종료 버튼. close_reason 선택 필요
- **JD 편집**: 프로젝트 메타 수정

## 4. Level 3 — 후보자 상세

후보자 행을 클릭하면 이 후보자의 모든 Application을 한눈에. **M:N 관계가 자연스럽게 드러나는 화면**.

### 레이아웃

```
┌────────────────────────────────────────────────────────┐
│ 김철수                                                  │
│ 1992년생 · 서울대 공대 · 5년차                         │
│ 현대차 → 네이버 → 카카오                                │
└────────────────────────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
이 후보자의 모든 Application (3건)

┌────────────────────────────────────────────────────────┐
│ [삼성전자 | AI Engineer]                               │
│ 발굴 ●○○○○  진행 18일  [보기 →]                      │
│ 최근 활동: 2026-04-12 이력서 수신                      │
├────────────────────────────────────────────────────────┤
│ [네이버 | ML Researcher]                               │
│ 추천 ●●●●○  진행 45일  [보기 →]                      │
│ 최근 활동: 2026-03-28 클라이언트 피드백 대기            │
├────────────────────────────────────────────────────────┤
│ [카카오 | 데이터 엔지니어]  (드롭됨)                    │
│ 드롭 사유: 이직의사 없음 (50일 전)                      │
│ [보기 →]                                               │
└────────────────────────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
커리어 히스토리 / 경력 / 학력 / ...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
미팅 기록 (PRE_MEETING 단계 자료)
- 2026-03-15 네이버 ML 포지션 사전미팅 (녹음 30분)
```

### 핵심 포인트

- 한 사람이 동시에 여러 프로젝트 파이프라인에 있다는 사실이 시각적으로 드러남
- 각 Application에서 "보기" 버튼 클릭 시 해당 Level 2로 이동 (해당 후보자 행에 스크롤 + 하이라이트)
- 후보자 자체의 커리어 정보는 하단에 (기존 synco candidate detail 재활용 가능)

## 5. Timeline 뷰 (보너스 — 설계 단계에서 개념만)

`ProjectEvent` 타임라인을 활용해서 프로젝트 히스토리를 시계열로 보여주는 뷰.

### 위치
Level 2 프로젝트 상세 페이지 내 탭 or 사이드 패널

### 예시

```
프로젝트 히스토리

┌─ 2026-04-14 10:23  phase_changed
│   사전미팅 → 서칭 (박정일)
│   사유: 김철수 추가로 인한 자동 후퇴
│
├─ 2026-04-14 10:20  application_added
│   김철수 (박정일)
│
├─ 2026-04-12 15:40  application_promoted
│   이영희 (검토 → 사전미팅)
│
├─ 2026-04-10 09:15  application_dropped
│   최하영 (부적합_경력)
│
├─ ...
│
└─ 2026-03-27 09:00  project_created
    박정일
```

이 뷰가 있으면 "이 프로젝트 왜 phase가 뒤로 갔지?" 같은 의문을 자동으로 해결.

## 6. 필요한 HTMX 엔드포인트 (개략)

```
GET  /projects/                         # Level 1 칸반 (현재 project_list 재설계)
GET  /projects/<id>/                    # Level 2 프로젝트 상세
GET  /projects/<id>/applications/       # 후보자 리스트 partial (hx-get)
POST /applications/<id>/promote/        # 다음 단계
POST /applications/<id>/drop/           # 드롭 (reason, note)
POST /applications/<id>/restore/        # 복구 (드롭 취소)
POST /projects/<id>/add_candidate/      # 후보자 추가 (candidate_id)
GET  /candidates/<id>/                  # Level 3 후보자 상세 (기존 재활용)
GET  /projects/<id>/timeline/           # ProjectEvent 타임라인 partial
POST /projects/<id>/close/              # 수동 프로젝트 종료
```

## 7. 기존 뷰·템플릿 영향 범위

### 교체 대상 (완전 재작성)
- `templates/projects/list.html` — 칸반 구조
- `templates/projects/partials/view_filters.html` — phase 기반 필터
- `templates/projects/detail.html` — Application 리스트 + 프로그레스 바
- `projects/views.py`의 list/detail/status_update — 전부 재작성

### 신규 작성
- `templates/projects/partials/application_row.html` — Application 한 행
- `templates/projects/partials/progress_bar.html` — 5단계 프로그레스
- `templates/projects/partials/drop_modal.html` — 드롭 사유 선택 모달
- `templates/projects/partials/add_candidate_modal.html` — 후보자 추가 모달
- `templates/projects/timeline.html` — 히스토리 타임라인

### 유지 (큰 변경 없음)
- JD 관련 업로드/편집 템플릿
- 승인 관련 (ProjectApproval)
- Candidate 자체 상세 페이지 (Level 3은 일부 추가)

## 8. 디자인 원칙

1. **컨설턴트가 직접 Phase를 만지지 못하게 한다.** UI에 phase 이동 버튼 없음. 모든 것은 Application 액션의 파생.
2. **Promote/Drop 버튼은 항상 2개만.** 복잡한 상태 선택 없음. 컨설턴트 인지 부담 최소화.
3. **드롭 사유는 반드시 enum에서 선택.** 자유 텍스트 말고 분류 체계로. 통계가 가능해짐.
4. **프로그레스 바가 1차 정보**. 컨설턴트가 한눈에 후보자 진척도를 파악할 수 있어야 함.
5. **M:N 관계를 숨기지 않는다.** 후보자 상세에서 모든 Application을 동시에 보여줌.
6. **Timeline은 부수적.** 메인 뷰는 현재 상태, 히스토리는 별도 탭/패널.

## 9. 향후 고려 — 스케일되면 추가할 기능

- **벌크 액션**: 여러 Application을 동시에 promote/drop (체크박스 선택 + 일괄 버튼)
- **알림/리마인더**: 특정 stage에서 N일 이상 멈춰있는 Application에 리마인더
- **필터 저장**: 자주 쓰는 필터 조합 저장
- **Application 간 비교**: 여러 후보자를 나란히 비교하는 뷰
- **Slack/Telegram 통합**: 주요 이벤트를 알림으로
- **주간 리포트**: 이번 주 promote/drop 집계, drop_reason 분포
