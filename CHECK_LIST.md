# 개발 체크리스트

코드 수정/기능 추가 시 반드시 점검할 항목들.

---

## 1. 의존성 & 환경 동일성

- [ ] 새 Python 패키지 추가 시 `pyproject.toml`에 명시했는가
- [ ] `uv lock` 실행하여 `uv.lock` 갱신했는가
- [ ] Dockerfile에서 설치되는 패키지와 로컬 환경이 동일한가
- [ ] 개발에서 되는 것이 Docker 컨테이너에서도 되는지 확인했는가
- [ ] `.env.example`에 새 환경변수 키를 추가했는가

## 2. UX 피드백

- [ ] 0.5초 이상 걸리는 작업에 로딩 상태(스피너, 버튼 비활성화)가 있는가
- [ ] 에러 발생 시 사용자에게 명확한 메시지를 보여주는가
- [ ] 성공/실패 후 다음 행동이 명확한가 (리다이렉트, 토스트 등)

## 3. 핵심 기능 노출

- [ ] 자주 쓰는 기능이 메뉴 안에 숨겨져 있지 않은가
- [ ] 대시보드/목록에서 주요 액션까지의 탭 수가 2회 이내인가
- [ ] 네비게이션 흐름에 빠진 중간 단계가 없는가 (목록 건너뛰고 바로 폼 진입 등)

## 4. 엣지 케이스 대응

- [ ] 사용자 입력이 예상과 다를 때 (빈 값, 이상한 형식) 처리가 되는가
- [ ] 파일 업로드 시 다양한 형식/구조를 고려했는가 (헤더 없음, 멀티시트 등)
- [ ] 하드코딩 규칙 대신 AI 판단에 위임할 수 있는 부분은 없는가

## 5. DB & 마이그레이션

- [ ] 모델 변경 후 `makemigrations` → `migrate` 실행했는가
- [ ] `makemigrations --check --dry-run`으로 누락된 migration이 없는가
- [ ] migration 파일이 git에 포함되어 있는가

## 6. 배포 전 확인

- [ ] `python manage.py check` 통과하는가
- [ ] Docker 이미지 빌드 (`docker compose up -d --build`) 성공하는가
- [ ] 컨테이너 안에서 핵심 기능 (AI 호출 등) 동작하는가
- [ ] 운영 DB 백업을 했는가
